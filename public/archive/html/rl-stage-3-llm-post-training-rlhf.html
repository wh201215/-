强化学习大模型路线｜Stage 3：LLM 后训练 / SFT / Reward Model / RLHF-PPO 数学推导

学习定位：Stage 1 解决了 MDP、Bellman、Q-learning。Stage 2 解决了 Policy Gradient、Actor-Critic、GAE、PPO。Stage 3 开始进入真正的大模型后训练：SFT → Reward Model → RLHF-PPO。

这一阶段你要建立一个核心理解：

\[ \text{LLM 后训练} \neq \text{普通监督微调} \]

它是把语言模型看作策略：

\[ \pi_\theta(y\mid x) \]

然后用人类偏好、奖励模型或规则奖励去优化这个策略，使模型更符合任务目标。

Stage 3 目标：

  1. 理解 SFT 的数学本质是条件语言建模；
  2. 理解偏好数据 chosen / rejected 的结构；
  3. 推导 Reward Model 的 Bradley-Terry loss；
  4. 理解 RLHF 为什么是 KL-constrained policy optimization；
  5. 把 Stage 2 的 PPO 公式放到大模型 token 生成中；
  6. 理解 reference model、KL penalty、value head 和 reward shaping。

1. LLM 后训练总流程

大模型从预训练到对齐，通常可以简化成四个阶段： 

Pretraining 预训练
  ↓
SFT 监督微调
  ↓
Reward Model 奖励模型训练
  ↓
RLHF-PPO 强化学习对齐
阶段 训练数据 训练目标 学到什么
Pretraining 大规模文本 预测下一个 token 语言、知识、基本推理模式
SFT prompt-response 指令数据 模仿高质量回答 对话格式、指令遵循、回答风格
Reward Model chosen / rejected 偏好对 学会判断哪个回答更好 人类偏好评分函数
RLHF-PPO 模型自己生成的回答 最大化 reward，同时限制偏移 更符合偏好的策略
一句话理解： SFT 让模型“会回答”；Reward Model 让系统“能评价回答”；RLHF 让模型“朝高评价回答的方向优化”。

1.1 为什么 SFT 之后还要 RLHF？

SFT 本质是模仿学习。它只告诉模型：

给定这个 prompt，应该模仿这个标准 response。

但它没有直接告诉模型：

  • 两个都看起来不错的回答，哪一个更符合人类偏好；
  • 回答太长、太短、太啰嗦、太武断时应该怎么权衡；
  • 模型生成自己没见过的回答时，怎样通过反馈继续变好；
  • 如何优化“帮助性、有用性、安全性、可读性”等非唯一标准。
关键差别： SFT 学的是“模仿给定答案”；RLHF 学的是“在很多可能答案中，让更受偏好的答案概率更高”。

2. LLM 为什么是策略 \(\pi_\theta\)

Stage 2 里，策略写成：

\[ \pi_\theta(a\mid s) \]

在语言模型中，我们把 prompt 和已经生成的 token 看作状态，把下一个 token 看作动作。

强化学习 语言模型
状态 \(s_t\) prompt + 已经生成的 token：\((x,y_{\lt t})\)
动作 \(a_t\) 下一个 token：\(y_t\)
策略 \(\pi_\theta(a_t\mid s_t)\) 语言模型概率：\(\pi_\theta(y_t\mid x,y_{\lt t})\)
轨迹 \(\tau\) 完整 response：\(y=(y_1,\ldots,y_T)\)
奖励 \(R\) 回答质量分数、偏好分数、规则分数

2.1 整段回答的概率

给定 prompt \(x\)，模型生成回答：

\[ y=(y_1,y_2,\ldots,y_T) \]

整段回答的概率由自回归分解给出：

\[ \pi_\theta(y\mid x) = \prod_{t=1}^{T} \pi_\theta(y_t\mid x,y_{\lt t}) \]

取 log：

\[ \log \pi_\theta(y\mid x) = \sum_{t=1}^{T} \log \pi_\theta(y_t\mid x,y_{\lt t}) \]
非常重要： 大模型里的策略更新通常会落到 token-level log probability 上，因为整段回答的 log 概率就是每个 token 的 log 概率之和。

3. 从预训练目标到 SFT

预训练和 SFT 的数学形式很像，都是语言建模。

3.1 预训练目标

给定一段文本 token：

\[ w=(w_1,w_2,\ldots,w_n) \]

语言模型学习：

\[ p_\theta(w_t\mid w_{\lt t}) \]

最大化整段文本概率：

\[ \max_\theta \sum_{t=1}^{n} \log p_\theta(w_t\mid w_{\lt t}) \]

等价于最小化负对数似然：

\[ L_{\mathrm{pretrain}}(\theta) = - \sum_{t=1}^{n} \log p_\theta(w_t\mid w_{\lt t}) \]

3.2 SFT 目标

SFT 数据不是普通连续文本，而是指令格式： 

User:
请解释 Bellman 方程。

Assistant:
Bellman 方程描述当前状态价值与下一状态价值之间的递推关系……

把用户输入记作 \(x\)，把助手回答记作 \(y\)：

\[ (x,y) \]

SFT 的目标是最大化：

\[ \pi_\theta(y\mid x) \]

也就是给定 prompt \(x\)，生成标准回答 \(y\) 的概率。

4. SFT Loss 逐步推导

4.1 单条样本

给定一条 SFT 样本：

\[ (x,y) \]

其中：

\[ y=(y_1,y_2,\ldots,y_T) \]

模型生成整段回答的概率是：

\[ \pi_\theta(y\mid x) = \prod_{t=1}^{T} \pi_\theta(y_t\mid x,y_{\lt t}) \]

4.2 最大似然目标

我们希望标准回答概率越大越好：

\[ \max_\theta \log \pi_\theta(y\mid x) \]

代入自回归展开：

\[ \max_\theta \sum_{t=1}^{T} \log \pi_\theta(y_t\mid x,y_{\lt t}) \]

4.3 写成最小化 loss

深度学习框架通常最小化 loss，所以取负号：

\[ \boxed{ L_{\mathrm{SFT}}(\theta) = - \sum_{t=1}^{T} \log \pi_\theta(y_t\mid x,y_{\lt t}) } \]

4.4 数据集形式

如果数据集是：

\[ \mathcal{D}_{\mathrm{SFT}} = \{(x_i,y_i)\}_{i=1}^{N} \]

总损失是：

\[ \boxed{ L_{\mathrm{SFT}}(\theta) = - \frac{1}{N} \sum_{i=1}^{N} \sum_{t=1}^{T_i} \log \pi_\theta(y_{i,t}\mid x_i,y_{i,\lt t}) } \]
一句话理解 SFT： 它就是让模型在 prompt 条件下，把标准回答中的每个 token 概率调高。

5. Prompt Masking 为什么重要

SFT 训练时，完整输入通常是： 

[System Prompt] + [User Prompt] + [Assistant Response]

但是 loss 一般只计算 Assistant Response 部分，不计算 User Prompt 部分。

5.1 为什么不训练模型预测用户输入？

因为我们的目标不是让模型学会复述用户问题，而是学会在用户问题后生成助手回答。

部分 是否计算 loss 原因
System Prompt 通常不计算 它是条件，不是要模型生成的目标
User Prompt 通常不计算 它是用户输入，不是助手输出
Assistant Response 计算 这是模型要学习生成的内容

5.2 Mask 后的 SFT Loss

设 \(m_t\) 是 mask：

\[ m_t= \begin{cases} 1, & \text{token } t \text{ 属于 assistant response}\\ 0, & \text{token } t \text{ 属于 prompt} \end{cases} \]

则 loss 写成：

\[ L_{\mathrm{SFT}}(\theta) = - \sum_{t=1}^{T} m_t \log \pi_\theta(z_t\mid z_{\lt t}) \]

其中 \(z\) 是拼接后的完整序列。

实践坑： 如果没有正确 mask prompt，模型可能浪费大量训练信号去预测用户输入格式，而不是学习高质量回答。

6. Preference Data 偏好数据

RLHF 不直接从标准答案学习，而是从偏好比较学习。

一条偏好数据通常是：

\[ (x,y_w,y_l) \]
符号 含义
\(x\) prompt
\(y_w\) winner / chosen response，更受偏好的回答
\(y_l\) loser / rejected response，不受偏好的回答

例如： 

Prompt:
请解释什么是强化学习。

Chosen:
强化学习研究智能体如何通过与环境交互，根据奖励信号学习长期最优行为。

Rejected:
强化学习就是一种机器学习，挺有用的。
偏好数据不是告诉模型唯一标准答案，而是告诉模型：在两个回答中，哪个更好。

6.1 为什么用比较而不是绝对打分？

人类给绝对分数很不稳定。比如同一个回答，一个标注者可能给 7 分，另一个可能给 8 分。

但比较两个回答哪个更好，相对更容易。

方式 难度 稳定性
给单个回答打 1-10 分 较难 不同人尺度差异大
比较 A 和 B 哪个更好 较容易 相对更稳定

7. Bradley-Terry 偏好概率模型

Reward Model 的训练通常基于 Bradley-Terry 模型。

7.1 Reward Model 输出什么？

Reward Model 接收 prompt 和 response，输出一个标量分数：

\[ r_\phi(x,y)\in \mathbb{R} \]

其中 \(\phi\) 是 Reward Model 参数。

分数越高，表示回答越好。

7.2 两个回答的偏好概率

给定两个回答 \(y_w\) 和 \(y_l\)，Bradley-Terry 模型假设：

\[ P(y_w \succ y_l\mid x) = \sigma \left( r_\phi(x,y_w)-r_\phi(x,y_l) \right) \]

其中 \(\sigma\) 是 sigmoid 函数：

\[ \sigma(z)=\frac{1}{1+\exp(-z)} \]

7.3 为什么用分数差？

我们关心的不是单个回答的绝对分数，而是 chosen 是否比 rejected 分数更高。

如果：

\[ r_\phi(x,y_w)-r_\phi(x,y_l) \gg 0 \]

则：

\[ \sigma(r_w-r_l)\approx 1 \]

表示模型非常确信 \(y_w\) 比 \(y_l\) 好。

如果：

\[ r_\phi(x,y_w)-r_\phi(x,y_l)=0 \]

则：

\[ \sigma(0)=0.5 \]

表示模型认为两者差不多。

一句话理解： Reward Model 不需要知道“绝对好到几分”，它只需要学会让 chosen 的分数高于 rejected。

8. Reward Model Loss 逐步推导

8.1 偏好样本的目标

对于一条偏好样本：

\[ (x,y_w,y_l) \]

我们希望：

\[ P(y_w \succ y_l\mid x) \]

尽可能接近 1。

8.2 写出似然

\[ P(y_w \succ y_l\mid x) = \sigma(r_\phi(x,y_w)-r_\phi(x,y_l)) \]

最大化这个概率等价于最大化 log 概率：

\[ \log \sigma(r_\phi(x,y_w)-r_\phi(x,y_l)) \]

8.3 写成最小化 loss

取负号：

\[ \boxed{ L_{\mathrm{RM}}(\phi) = - \log \sigma \left( r_\phi(x,y_w)-r_\phi(x,y_l) \right) } \]

8.4 数据集形式

如果偏好数据集为：

\[ \mathcal{D}_{\mathrm{pref}} = \{(x_i,y_{i,w},y_{i,l})\}_{i=1}^{N} \]

则：

\[ \boxed{ L_{\mathrm{RM}}(\phi) = - \frac{1}{N} \sum_{i=1}^{N} \log \sigma \left( r_\phi(x_i,y_{i,w}) - r_\phi(x_i,y_{i,l}) \right) } \]

8.5 梯度直观理解

如果 Reward Model 给 chosen 的分数不够高，或者给 rejected 的分数过高，loss 就会变大。

情况 loss 倾向 训练会怎么调整
\(r_w \gg r_l\) loss 小 说明判断正确
\(r_w = r_l\) loss 中等 推动 \(r_w\) 变大、\(r_l\) 变小
\(r_w < r_l\) loss 大 强烈纠正分数顺序
Reward Model 训练结束后： 它就可以给模型自己生成的回答打分，作为 RLHF 的奖励信号。

9. RLHF 的优化目标

训练好 Reward Model 后，我们让当前语言模型生成回答：

\[ y\sim \pi_\theta(\cdot\mid x) \]

然后 Reward Model 给分：

\[ r_\phi(x,y) \]

最直观的目标是：

\[ \max_\theta \mathbb{E}_{x\sim \mathcal{D},y\sim \pi_\theta(\cdot\mid x)} [ r_\phi(x,y) ] \]

但是如果只最大化 reward，会有问题。模型可能为了骗 Reward Model，偏离原本语言模型太远。

9.1 加入 KL 约束

因此 RLHF 的常见目标是：

\[ \boxed{ \max_\theta \mathbb{E}_{x,y\sim \pi_\theta} \left[ r_\phi(x,y) - \beta \mathrm{KL} \left( \pi_\theta(\cdot\mid x) \| \pi_{\mathrm{ref}}(\cdot\mid x) \right) \right] } \]
符号 含义
\(\pi_\theta\) 正在训练的 policy model
\(\pi_{\mathrm{ref}}\) reference model，通常是 SFT model 的冻结副本
\(r_\phi(x,y)\) Reward Model 给出的分数
\(\beta\) KL 惩罚系数
一句话理解： RLHF 不是单纯追求 Reward Model 高分，而是在“高 reward”和“不偏离原模型太远”之间折中。

10. KL 约束为什么必须有

KL divergence 衡量两个概率分布之间的差异。

10.1 KL 定义

\[ \mathrm{KL}(P\|Q) = \sum_x P(x) \log \frac{P(x)}{Q(x)} \]

在大模型里，可以理解成：

\[ \mathrm{KL} \left( \pi_\theta(\cdot\mid s) \| \pi_{\mathrm{ref}}(\cdot\mid s) \right) = \sum_a \pi_\theta(a\mid s) \log \frac{ \pi_\theta(a\mid s) }{ \pi_{\mathrm{ref}}(a\mid s) } \]

10.2 KL 惩罚的作用

没有 KL 有 KL
模型可能为了高 reward 输出奇怪文本 模型被限制在原有语言能力附近
容易 reward hacking 降低奖励投机风险
训练不稳定 训练更稳
可能丢失 SFT 学到的对话能力 保留原本指令遵循和语言质量
关键理解： Reward Model 不完美。如果没有 KL，Policy Model 可能学会利用 Reward Model 的漏洞，而不是变得真正更好。

10.3 \(\beta\) 怎么理解？

\[ r_{\mathrm{total}} = r_\phi(x,y) - \beta \cdot \mathrm{KL} \]
\(\beta\) 效果 风险
太小 模型更大胆追求 reward 容易跑偏、reward hacking
太大 模型更接近 reference model 学不动、提升有限
适中 reward 和稳定性平衡 实践中需要调参

11. LLM-PPO 如何使用 Stage 2 的 PPO

Stage 2 的 PPO 目标是：

\[ L^{\mathrm{CLIP}}(\theta) = \mathbb{E}_t \left[ \min \left( r_t(\theta)\hat{A}_t, \mathrm{clip}(r_t(\theta),1-\epsilon,1+\epsilon)\hat{A}_t \right) \right] \]

其中：

\[ r_t(\theta) = \frac{ \pi_\theta(A_t\mid S_t) }{ \pi_{\theta_{\mathrm{old}}}(A_t\mid S_t) } \]

11.1 放到 LLM 中

LLM 的状态是：

\[ S_t=(x,y_{\lt t}) \]

动作是：

\[ A_t=y_t \]

所以 ratio 是：

\[ r_t(\theta) = \frac{ \pi_\theta(y_t\mid x,y_{\lt t}) }{ \pi_{\theta_{\mathrm{old}}}(y_t\mid x,y_{\lt t}) } \]

这就是新模型和旧模型对同一个 token 的概率比。

11.2 为什么 PPO 是 token-level

因为整段回答的概率是 token 概率连乘：

\[ \pi_\theta(y\mid x) = \prod_{t=1}^{T} \pi_\theta(y_t\mid x,y_{\lt t}) \]

如果写整段 ratio：

\[ \frac{\pi_\theta(y\mid x)} {\pi_{\theta_{\mathrm{old}}}(y\mid x)} = \prod_{t=1}^{T} \frac{ \pi_\theta(y_t\mid x,y_{\lt t}) }{ \pi_{\theta_{\mathrm{old}}}(y_t\mid x,y_{\lt t}) } \]

整段 ratio 容易数值不稳定，所以实践中通常对每个 token 的 logprob、ratio、advantage 做处理。

大模型 PPO 的核心： Reward 往往是整段回答级别的，但策略概率是 token-level 的，所以训练时要把 sequence-level reward 转化为 token-level 更新信号。

12. Reward Shaping 与 KL Penalty

在 LLM-RLHF 里，最终 reward 通常来自两部分：

奖励来源 含义
Reward Model score 整段回答质量分数
KL penalty 每个 token 偏离 reference model 的惩罚

12.1 每个 token 的 KL 惩罚

对于生成的 token \(y_t\)，常用近似 KL penalty：

\[ \mathrm{kl}_t = \log \pi_\theta(y_t\mid x,y_{\lt t}) - \log \pi_{\mathrm{ref}}(y_t\mid x,y_{\lt t}) \]

如果当前模型比 reference model 更倾向于生成这个 token，则 \(\mathrm{kl}_t\) 可能较大。

12.2 shaped reward

每个 token 可以得到一个 KL 惩罚：

\[ r_t^{\mathrm{KL}} = -\beta \left[ \log \pi_\theta(y_t\mid x,y_{\lt t}) - \log \pi_{\mathrm{ref}}(y_t\mid x,y_{\lt t}) \right] \]

最终 token 的 reward 可以写成：

\[ r_t= r_t^{\mathrm{KL}} \quad (t

在最后一个 token 或 response 结束位置，加上 Reward Model 分数：

\[ r_T= r_T^{\mathrm{KL}} + r_\phi(x,y) \]
一句话理解 reward shaping： 整段回答结束时给质量奖励；每个 token 生成时都给 KL 惩罚，防止模型偏离 reference model。

13. Value Head 和 Advantage

PPO 需要 advantage：

\[ \hat{A}_t \]

为了估计 advantage，LLM-PPO 通常在语言模型顶部加一个 value head：

\[ V_\psi(x,y_{\lt t}) \]

它表示：当前已经生成到 \(t\) 步时，后面预计还能得到多少 reward。

13.1 Value Head 输入是什么？

传统 RL LLM-PPO
\(V(s_t)\) \(V_\psi(x,y_{\lt t})\)
状态价值 当前 prompt 和已生成 token 的价值

13.2 Value Loss

Value head 的训练目标是拟合 return：

\[ L_V(\psi) = \frac{1}{2} \sum_t \left( V_\psi(x,y_{\lt t}) - \hat{R}_t \right)^2 \]

其中 \(\hat{R}_t\) 是从第 \(t\) 步开始的 reward-to-go。

13.3 Advantage 估计

最简单可以写成：

\[ \hat{A}_t = \hat{R}_t - V_\psi(x,y_{\lt t}) \]

更常见的是用 GAE：

\[ \hat{A}_t^{\mathrm{GAE}} = \sum_{l=0}^{\infty} (\gamma\lambda)^l \delta_{t+l} \]

其中：

\[ \delta_t = r_t + \gamma V_\psi(x,y_{\le t}) - V_\psi(x,y_{\lt t}) \]
LLM 中的 Critic： Value head 就是 Stage 2 里 Critic 的大模型版本。

14. Token-level Credit Assignment

大模型 RLHF 有一个难点：奖励通常在整段回答结束后才出现。 

Prompt:
请证明某个结论。

Model response:
token1 token2 token3 ... tokenT

Reward Model:
整段 response 得分 2.7

问题是：

这 2.7 分应该分给哪些 token？是所有 token 都有贡献，还是只有最后结论有贡献？

14.1 简单处理：所有 token 共享最终 reward

最简单做法是用同一个最终 reward 影响整段生成。

这类似 Stage 2 中 REINFORCE 的思想：

\[ \nabla_\theta J \approx \sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(y_t\mid x,y_{\lt t}) \cdot \hat{A}_t \]

如果整段回答 reward 高，就提高这段回答中 token 的概率。

14.2 问题：长回答信用分配更难

如果回答很长，其中某些 token 可能是错误的，但最终 reward 只有一个标量，就很难准确判断哪一步导致了好或坏。

奖励粒度 优点 缺点
Sequence-level reward 容易标注和训练 Reward Model 信用分配粗糙
Token-level reward 信用分配细 标注和建模困难
Process reward 适合推理任务 需要过程级验证或标注
后面学 GRPO / RLVR 时要记住： 数学和代码任务之所以适合 RL，是因为它们可以通过最终答案或测试结果自动给 reward，但长链推理仍然存在 credit assignment 难题。

15. RLHF-PPO 算法流程

现在把整个流程串起来。

15.1 第一步：准备 SFT 模型

Base Model
  ↓ SFT
SFT Model

SFT Model 既作为 RLHF 的初始化模型，也常作为 reference model。

15.2 第二步：训练 Reward Model

Prompt x
  ↓
Response A
Response B
  ↓
人类偏好标注
  ↓
chosen / rejected
  ↓
训练 Reward Model

Reward Model loss：

\[ L_{\mathrm{RM}} = - \log \sigma \left( r_\phi(x,y_w)-r_\phi(x,y_l) \right) \]

15.3 第三步：PPO Rollout

用当前 policy model 生成回答：

\[ y\sim \pi_\theta(\cdot\mid x) \]

记录每个 token 的 logprob：

\[ \log \pi_\theta(y_t\mid x,y_{\lt t}) \]

同时计算 reference logprob：

\[ \log \pi_{\mathrm{ref}}(y_t\mid x,y_{\lt t}) \]

15.4 第四步：计算 reward

\[ r_{\mathrm{RM}}=r_\phi(x,y) \]
\[ r_t^{\mathrm{KL}} = -\beta \left[ \log \pi_\theta(y_t\mid x,y_{\lt t}) - \log \pi_{\mathrm{ref}}(y_t\mid x,y_{\lt t}) \right] \]

最终得到每个 token 的 shaped reward。

15.5 第五步：计算 advantage

用 value head 和 GAE 计算：

\[ \hat{A}_t \]

15.6 第六步：PPO 更新

\[ r_t(\theta) = \frac{ \pi_\theta(y_t\mid x,y_{\lt t}) }{ \pi_{\theta_{\mathrm{old}}}(y_t\mid x,y_{\lt t}) } \]
\[ L^{\mathrm{CLIP}} = \mathbb{E}_t \left[ \min \left( r_t(\theta)\hat{A}_t, \mathrm{clip}(r_t(\theta),1-\epsilon,1+\epsilon)\hat{A}_t \right) \right] \]

15.7 总流程伪代码

初始化 policy model = SFT model
初始化 reference model = frozen SFT model
训练好 reward model

for each RLHF iteration:

    1. 从 prompt dataset 采样一批 prompts

    2. policy model 生成 responses

    3. reward model 给每个 response 打分

    4. 计算 policy model 与 reference model 的 KL penalty

    5. 得到 shaped rewards

    6. value head 估计 values

    7. 用 GAE 计算 advantages

    8. 用 PPO clip objective 更新 policy model

    9. 更新 value head
这一节要背下来： RLHF-PPO 的核心不是“拿 Reward Model 分数直接微调”，而是“生成样本 → 打分 → 计算 advantage → PPO 限制更新 → 保持 KL 稳定”。

16. 常见坑

16.1 Reward Hacking

Reward Model 不完美。Policy Model 可能找到让 Reward Model 打高分但人类不喜欢的回答。

目标 Reward Model 漏洞 模型可能学到
有帮助 偏好长回答 输出冗长废话
有礼貌 偏好客套表达 过度道歉、空洞客气
推理充分 偏好长推理 编造复杂但错误的推理
安全 过度惩罚风险 过度拒答

16.2 KL 太小或太大

问题 表现
KL 系数太小 模型偏离太快，输出风格崩坏
KL 系数太大 模型几乎不变，reward 提升有限

16.3 Value Head 不准

如果 value head 估计不准，advantage 就会很噪，PPO 更新会不稳定。

16.4 Reward Scale 不稳定

Reward Model 输出分数尺度如果变化太大，PPO 的 advantage 也会不稳定。

实践判断： RLHF-PPO 难不是因为公式特别难，而是因为 reward、KL、value、advantage、采样分布互相影响，训练系统非常敏感。

17. 为什么下一阶段要学 DPO

RLHF-PPO 很强，但工程复杂。它需要：

  • 训练 Reward Model；
  • 维护 reference model；
  • 做在线 rollout；
  • 训练 value head；
  • 计算 advantage；
  • 调 PPO clip、KL、reward scale；
  • 防止 reward hacking。

DPO 的出现，就是为了简化这个过程。

Stage 4 的核心问题： 既然我们最终还是从 chosen / rejected 偏好数据学习，能不能不显式训练 Reward Model，也不用 PPO，直接优化 policy？

这就引出 Stage 4： 

DPO Direct Preference Optimization
直接偏好优化

它会把 RLHF 的 KL-constrained reward maximization 推导成一个非常简洁的偏好分类 loss。

18. 一页总结

模块 核心公式 你要理解什么
LLM 策略 \(\pi_\theta(y\mid x)=\prod_t\pi_\theta(y_t\mid x,y_{\lt t})\) 整段回答概率由 token 概率连乘。
SFT Loss \(L_{\mathrm{SFT}}=-\sum_t\log\pi_\theta(y_t\mid x,y_{\lt t})\) 提高标准回答 token 的概率。
偏好数据 \((x,y_w,y_l)\) chosen 比 rejected 更受偏好。
Reward Model \(r_\phi(x,y)\) 给 prompt-response 打一个标量分数。
BT 概率 \(\sigma(r_w-r_l)\) chosen 胜过 rejected 的概率。
RM Loss \(-\log\sigma(r_w-r_l)\) 让 chosen 分数高于 rejected。
RLHF 目标 \(\mathbb{E}[r_\phi-\beta KL]\) 高 reward 和不偏离 reference 之间折中。
Token Ratio \(\frac{\pi_\theta(y_t\mid x,y_{\lt t})}{\pi_{\mathrm{old}}(y_t\mid x,y_{\lt t})}\) 新旧模型对同一 token 的概率变化。
Value Head \(V_\psi(x,y_{\lt t})\) 估计当前生成状态的未来 reward。
PPO Clip \(\min(r_t\hat{A}_t,\mathrm{clip}(r_t,1-\epsilon,1+\epsilon)\hat{A}_t)\) 限制 policy 每次更新不要太猛。
进入 Stage 4 前的标准： 你应该能解释清楚：
  1. SFT loss 为什么是 token-level negative log likelihood；
  2. Reward Model 为什么用 \(-\log\sigma(r_w-r_l)\)；
  3. RLHF 为什么要加 KL penalty；
  4. LLM-PPO 中 ratio 是新旧模型对同一个 token 的概率比；
  5. 为什么 PPO 工程复杂，DPO 才有必要出现。

19. 本阶段练习

19.1 手推练习

  1. 从 \(\pi_\theta(y\mid x)=\prod_t\pi_\theta(y_t\mid x,y_{\lt t})\) 推导 SFT loss。
  2. 从 Bradley-Terry 模型推导 Reward Model loss。
  3. 解释为什么 \(\sigma(r_w-r_l)\) 可以表示 chosen 胜过 rejected 的概率。
  4. 写出 RLHF 的 KL-constrained objective，并解释每一项。
  5. 把 PPO ratio 从传统 \(A_t,S_t\) 改写成 LLM token 形式。

19.2 编程练习

建议做一个不训练大模型的 toy demo：

1. 准备 5 条 prompt。
2. 为每条 prompt 写两个 response：chosen 和 rejected。
3. 假设 reward model 输出两个分数 r_w, r_l。
4. 手动计算：
   loss = -log sigmoid(r_w - r_l)
5. 改变 r_w - r_l，观察 loss 如何变化。
6. 再模拟一个 token-level KL penalty：
   kl_t = logp_policy_t - logp_ref_t
   reward_t = -beta * kl_t
