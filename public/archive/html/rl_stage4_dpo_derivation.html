强化学习大模型路线｜Stage 4：DPO 直接偏好优化数学推导

学习定位：Stage 3 讲了 SFT、Reward Model、RLHF-PPO。Stage 4 进入 DPO，Direct Preference Optimization，直接偏好优化。

DPO 的关键思想是：

\[ \text{不显式训练 Reward Model，不跑 PPO，也可以直接用偏好数据优化策略模型。} \]

你这一阶段要掌握的核心链条是：

RLHF 的 KL 约束目标
  ↓
求出最优策略 π*
  ↓
把 reward 写成 policy 和 reference policy 的 log ratio
  ↓
代入 Bradley-Terry 偏好模型
  ↓
得到 DPO loss

Stage 4 目标：

  1. 理解 DPO 为什么来自 RLHF，而不是一个普通分类 loss；
  2. 完整推导 DPO 的核心公式；
  3. 理解 reference model 在 DPO 中的作用；
  4. 理解 \(\beta\) 控制什么；
  5. 能解释 DPO 与 SFT、Reward Model、PPO 的区别；
  6. 为 Stage 5 的 GRPO / RLVR 做准备。

1. 为什么 RLHF-PPO 之后要学 DPO？

Stage 3 里的 RLHF-PPO 很完整，但工程链路比较重。 

SFT Model
  ↓
收集 chosen / rejected 偏好数据
  ↓
训练 Reward Model
  ↓
用当前 Policy 生成回答
  ↓
Reward Model 打分
  ↓
计算 KL penalty
  ↓
训练 Value Head
  ↓
计算 Advantage / GAE
  ↓
PPO 更新 Policy

这里面有很多组件：

组件 作用 复杂性
Reward Model 给回答打分 需要单独训练，可能 reward hacking
Policy Model 被优化的语言模型 需要 rollout 采样
Reference Model 提供 KL 约束 需要保存和计算 logprob
Value Head 估计 advantage 训练不稳会影响 PPO
PPO 策略优化 超参数多，训练敏感
RLHF-PPO 的痛点： 它不是数学上不能理解，而是工程上组件多、调参多、训练不稳定。

DPO 想解决的问题是：

既然我们本来就有 chosen / rejected 偏好数据，能不能直接用这些偏好数据优化语言模型，而不是先训练 Reward Model，再跑 PPO？

2. 回顾 RLHF 的 KL 约束目标

DPO 的推导从 Stage 3 的 RLHF 目标开始。

给定 prompt \(x\)，我们希望找到一个策略 \(\pi\)，让它生成的回答 \(y\) 得到高 reward，同时不要偏离 reference policy 太远。

\[ \max_{\pi} \mathbb{E}_{y\sim \pi(\cdot\mid x)} \left[ r(x,y) \right] - \beta \mathrm{KL} \left( \pi(\cdot\mid x) \| \pi_{\mathrm{ref}}(\cdot\mid x) \right) \]

其中：

符号 含义
\(\pi(\cdot\mid x)\) 我们要优化的策略
\(\pi_{\mathrm{ref}}(\cdot\mid x)\) reference policy，通常是 SFT 模型
\(r(x,y)\) 回答 \(y\) 的奖励
\(\beta\) 控制 KL 惩罚强度的系数

2.1 把 KL 展开

KL 定义为：

\[ \mathrm{KL} \left( \pi \| \pi_{\mathrm{ref}} \right) = \mathbb{E}_{y\sim \pi} \left[ \log \frac{ \pi(y\mid x) }{ \pi_{\mathrm{ref}}(y\mid x) } \right] \]

代入目标：

\[ \max_{\pi} \mathbb{E}_{y\sim \pi} \left[ r(x,y) - \beta \log \frac{ \pi(y\mid x) }{ \pi_{\mathrm{ref}}(y\mid x) } \right] \]

这就是 DPO 推导的起点。

一句话： RLHF 的目标不是单纯最大化 reward，而是最大化 reward，同时惩罚和 reference model 的差异。

3. 求解最优策略 \(\pi^*\)

现在我们固定 prompt \(x\)，把问题看成对所有可能回答 \(y\) 的分布优化。

目标函数写成：

\[ \max_{\pi} \sum_y \pi(y\mid x) \left[ r(x,y) - \beta \log \frac{ \pi(y\mid x) }{ \pi_{\mathrm{ref}}(y\mid x) } \right] \]

同时策略必须满足概率归一化：

\[ \sum_y \pi(y\mid x)=1 \]

3.1 使用拉格朗日乘子

构造拉格朗日函数：

\[ \mathcal{L}(\pi,\lambda) = \sum_y \pi(y\mid x) \left[ r(x,y) - \beta \log \frac{ \pi(y\mid x) }{ \pi_{\mathrm{ref}}(y\mid x) } \right] + \lambda \left( \sum_y \pi(y\mid x)-1 \right) \]

为了简化记号，下面暂时把 \(\pi(y\mid x)\) 写成 \(\pi_y\)，把 \(\pi_{\mathrm{ref}}(y\mid x)\) 写成 \(\pi_{\mathrm{ref},y}\)。

\[ \mathcal{L} = \sum_y \pi_y \left[ r_y - \beta \log \frac{\pi_y}{\pi_{\mathrm{ref},y}} \right] + \lambda \left( \sum_y \pi_y-1 \right) \]

3.2 对 \(\pi_y\) 求导

我们需要计算：

\[ \frac{\partial \mathcal{L}}{\partial \pi_y} \]

先看其中一项：

\[ \pi_y \left[ r_y - \beta \log \frac{\pi_y}{\pi_{\mathrm{ref},y}} \right] \]

展开：

\[ \pi_y r_y - \beta \pi_y \log \pi_y + \beta \pi_y \log \pi_{\mathrm{ref},y} \]

逐项求导：

\[ \frac{\partial}{\partial \pi_y}(\pi_y r_y)=r_y \]
\[ \frac{\partial}{\partial \pi_y} \left( -\beta \pi_y \log \pi_y \right) = -\beta(\log \pi_y+1) \]
\[ \frac{\partial}{\partial \pi_y} \left( \beta \pi_y \log \pi_{\mathrm{ref},y} \right) = \beta \log \pi_{\mathrm{ref},y} \]

拉格朗日约束项求导为：

\[ \frac{\partial}{\partial \pi_y} \lambda \left( \sum_y \pi_y-1 \right) = \lambda \]

合并：

\[ \frac{\partial \mathcal{L}}{\partial \pi_y} = r_y - \beta(\log \pi_y+1) + \beta\log \pi_{\mathrm{ref},y} + \lambda \]

3.3 令导数为 0

\[ r_y - \beta(\log \pi_y+1) + \beta\log \pi_{\mathrm{ref},y} + \lambda = 0 \]

整理：

\[ \beta\log \pi_y = r_y + \beta\log \pi_{\mathrm{ref},y} + \lambda - \beta \]

两边除以 \(\beta\)：

\[ \log \pi_y = \frac{1}{\beta}r_y + \log \pi_{\mathrm{ref},y} + \frac{\lambda}{\beta} - 1 \]

指数化：

\[ \pi_y = \pi_{\mathrm{ref},y} \exp \left( \frac{1}{\beta}r_y \right) \exp \left( \frac{\lambda}{\beta}-1 \right) \]

最后一项和 \(y\) 无关，是归一化常数的一部分。记作：

\[ \frac{1}{Z(x)} = \exp \left( \frac{\lambda}{\beta}-1 \right) \]

于是得到最优策略：

\[ \boxed{ \pi^*(y\mid x) = \frac{1}{Z(x)} \pi_{\mathrm{ref}}(y\mid x) \exp \left( \frac{1}{\beta}r(x,y) \right) } \]

3.4 这个公式怎么理解？

部分 含义
\(\pi_{\mathrm{ref}}(y\mid x)\) 原来的参考模型概率
\(\exp(r(x,y)/\beta)\) reward 越高，回答概率被放大越多
\(Z(x)\) 归一化常数，保证所有回答概率加起来等于 1
一句话理解： 最优策略等于 reference policy 乘上一个 reward 指数加权项。reward 高的回答会被提高概率，reward 低的回答会被降低概率。

4. 从最优策略反推出隐式奖励

刚才得到：

\[ \pi^*(y\mid x) = \frac{1}{Z(x)} \pi_{\mathrm{ref}}(y\mid x) \exp \left( \frac{1}{\beta}r(x,y) \right) \]

现在我们把它反过来，解出 \(r(x,y)\)。

4.1 两边乘以 \(Z(x)\)

\[ Z(x)\pi^*(y\mid x) = \pi_{\mathrm{ref}}(y\mid x) \exp \left( \frac{1}{\beta}r(x,y) \right) \]

4.2 两边除以 reference policy

\[ \frac{ Z(x)\pi^*(y\mid x) }{ \pi_{\mathrm{ref}}(y\mid x) } = \exp \left( \frac{1}{\beta}r(x,y) \right) \]

4.3 两边取 log

\[ \log Z(x) + \log \frac{ \pi^*(y\mid x) }{ \pi_{\mathrm{ref}}(y\mid x) } = \frac{1}{\beta}r(x,y) \]

4.4 两边乘以 \(\beta\)

\[ \boxed{ r(x,y) = \beta \log \frac{ \pi^*(y\mid x) }{ \pi_{\mathrm{ref}}(y\mid x) } + \beta \log Z(x) } \]

这一步是 DPO 的灵魂。

关键理解： 如果一个策略 \(\pi^*\) 是某个 reward 下的最优策略，那么这个 reward 可以用 \(\pi^*\) 和 reference policy 的 log ratio 表示出来。

4.5 为什么 \(\log Z(x)\) 后面会消掉？

偏好模型只关心两个回答的 reward 差：

\[ r(x,y_w)-r(x,y_l) \]

代入上式：

\[ r(x,y_w) = \beta \log \frac{ \pi^*(y_w\mid x) }{ \pi_{\mathrm{ref}}(y_w\mid x) } + \beta\log Z(x) \]
\[ r(x,y_l) = \beta \log \frac{ \pi^*(y_l\mid x) }{ \pi_{\mathrm{ref}}(y_l\mid x) } + \beta\log Z(x) \]

两者相减：

\[ r(x,y_w)-r(x,y_l) = \beta \log \frac{ \pi^*(y_w\mid x) }{ \pi_{\mathrm{ref}}(y_w\mid x) } - \beta \log \frac{ \pi^*(y_l\mid x) }{ \pi_{\mathrm{ref}}(y_l\mid x) } \]

因为两边都有同一个 \(\beta\log Z(x)\)，所以相减后消失。

这就是 DPO 不需要知道归一化常数 \(Z(x)\) 的原因。

5. 接入 Bradley-Terry 偏好模型

Stage 3 里我们学过 Reward Model 的偏好概率：

\[ P(y_w \succ y_l\mid x) = \sigma \left( r(x,y_w)-r(x,y_l) \right) \]

其中：

\[ \sigma(z)=\frac{1}{1+\exp(-z)} \]

现在我们不用显式 Reward Model，而是用上一节推出来的隐式 reward。

5.1 用 policy log ratio 表示 reward 差

由上一节：

\[ r(x,y_w)-r(x,y_l) = \beta \left[ \log \frac{ \pi^*(y_w\mid x) }{ \pi_{\mathrm{ref}}(y_w\mid x) } - \log \frac{ \pi^*(y_l\mid x) }{ \pi_{\mathrm{ref}}(y_l\mid x) } \right] \]

DPO 用当前要训练的模型 \(\pi_\theta\) 近似 \(\pi^*\)：

\[ \pi^*(y\mid x) \approx \pi_\theta(y\mid x) \]

于是：

\[ r_\theta(x,y_w)-r_\theta(x,y_l) = \beta \left[ \log \frac{ \pi_\theta(y_w\mid x) }{ \pi_{\mathrm{ref}}(y_w\mid x) } - \log \frac{ \pi_\theta(y_l\mid x) }{ \pi_{\mathrm{ref}}(y_l\mid x) } \right] \]

5.2 得到 DPO 偏好概率

\[ P_\theta(y_w \succ y_l\mid x) = \sigma \left( \beta \left[ \log \frac{ \pi_\theta(y_w\mid x) }{ \pi_{\mathrm{ref}}(y_w\mid x) } - \log \frac{ \pi_\theta(y_l\mid x) }{ \pi_{\mathrm{ref}}(y_l\mid x) } \right] \right) \]
一句话理解： DPO 判断 chosen 是否胜过 rejected，不是看原始概率谁大，而是看相对于 reference model，当前模型是否更偏向 chosen。

6. DPO Loss 完整推导

6.1 单条偏好样本

一条偏好样本是：

\[ (x,y_w,y_l) \]

其中 \(y_w\) 是 chosen，\(y_l\) 是 rejected。

我们希望模型认为：

\[ P_\theta(y_w \succ y_l\mid x) \]

越接近 1 越好。

6.2 最大化 log likelihood

\[ \max_\theta \log P_\theta(y_w \succ y_l\mid x) \]

代入上一节的概率：

\[ \max_\theta \log \sigma \left( \beta \left[ \log \frac{ \pi_\theta(y_w\mid x) }{ \pi_{\mathrm{ref}}(y_w\mid x) } - \log \frac{ \pi_\theta(y_l\mid x) }{ \pi_{\mathrm{ref}}(y_l\mid x) } \right] \right) \]

6.3 写成最小化 loss

取负号，得到 DPO loss：

\[ \boxed{ L_{\mathrm{DPO}}(\theta) = - \log \sigma \left( \beta \left[ \log \frac{ \pi_\theta(y_w\mid x) }{ \pi_{\mathrm{ref}}(y_w\mid x) } - \log \frac{ \pi_\theta(y_l\mid x) }{ \pi_{\mathrm{ref}}(y_l\mid x) } \right] \right) } \]

6.4 展开成 logprob 形式

用：

\[ \log \frac{a}{b}=\log a-\log b \]

得到：

\[ L_{\mathrm{DPO}}(\theta) = - \log \sigma \left( \beta \left[ \log\pi_\theta(y_w\mid x) - \log\pi_{\mathrm{ref}}(y_w\mid x) - \log\pi_\theta(y_l\mid x) + \log\pi_{\mathrm{ref}}(y_l\mid x) \right] \right) \]

也可以写成：

\[ \boxed{ L_{\mathrm{DPO}}(\theta) = - \log \sigma \left( \beta \left[ \log\pi_\theta(y_w\mid x) - \log\pi_\theta(y_l\mid x) - \log\pi_{\mathrm{ref}}(y_w\mid x) + \log\pi_{\mathrm{ref}}(y_l\mid x) \right] \right) } \]

6.5 数据集形式

如果偏好数据集是：

\[ \mathcal{D} = \{(x_i,y_{i,w},y_{i,l})\}_{i=1}^{N} \]

则：

\[ \boxed{ L_{\mathrm{DPO}}(\theta) = - \frac{1}{N} \sum_{i=1}^{N} \log \sigma \left( \beta \left[ \log \frac{ \pi_\theta(y_{i,w}\mid x_i) }{ \pi_{\mathrm{ref}}(y_{i,w}\mid x_i) } - \log \frac{ \pi_\theta(y_{i,l}\mid x_i) }{ \pi_{\mathrm{ref}}(y_{i,l}\mid x_i) } \right] \right) } \]
这一节是 Stage 4 最核心公式。 你要能从 RLHF 目标一路推到这个 loss，而不是只背结论。

7. DPO Loss 每一项是什么意思？

DPO 的核心量是：

\[ \Delta_\theta = \log \frac{ \pi_\theta(y_w\mid x) }{ \pi_{\mathrm{ref}}(y_w\mid x) } - \log \frac{ \pi_\theta(y_l\mid x) }{ \pi_{\mathrm{ref}}(y_l\mid x) } \]

DPO loss 就是：

\[ L_{\mathrm{DPO}} = -\log\sigma(\beta\Delta_\theta) \]

7.1 第一项：chosen 的 log ratio

\[ \log \frac{ \pi_\theta(y_w\mid x) }{ \pi_{\mathrm{ref}}(y_w\mid x) } \]

它表示：相对于 reference model，当前模型是否更愿意生成 chosen。

值 含义
大于 0 当前模型比 reference 更偏向 chosen
等于 0 当前模型和 reference 对 chosen 的倾向一样
小于 0 当前模型比 reference 更不偏向 chosen

7.2 第二项：rejected 的 log ratio

\[ \log \frac{ \pi_\theta(y_l\mid x) }{ \pi_{\mathrm{ref}}(y_l\mid x) } \]

它表示：相对于 reference model，当前模型是否更愿意生成 rejected。

7.3 两者相减

\[ \Delta_\theta = \text{chosen 相对提升} - \text{rejected 相对提升} \]

如果 \(\Delta_\theta\) 越大，说明当前模型相对于 reference，更偏向 chosen 而不是 rejected。

DPO 不是简单地让 chosen 概率大于 rejected 概率。 它让当前模型相对于 reference model，更加偏向 chosen。

7.4 为什么需要 reference model？

如果没有 reference model，目标可能退化成：

\[ -\log\sigma \left( \beta[ \log\pi_\theta(y_w\mid x) - \log\pi_\theta(y_l\mid x) ] \right) \]

这会直接鼓励 chosen 概率高于 rejected 概率，但缺少“不要偏离原模型太远”的约束来源。

reference model 的作用类似 Stage 3 里的 KL anchor：

在 RLHF-PPO 中 在 DPO 中
显式 KL penalty 限制 policy 偏移 通过 log ratio 隐式引入 reference policy
需要 rollout 和 PPO 直接在偏好数据上优化
一句话： reference model 让 DPO 不是无约束地追逐 chosen，而是在相对于原模型的变化中优化偏好。

8. DPO 梯度直观理解

DPO loss：

\[ L_{\mathrm{DPO}} = -\log\sigma(\beta\Delta_\theta) \]

其中：

\[ \Delta_\theta = \log\pi_\theta(y_w\mid x) - \log\pi_\theta(y_l\mid x) - \log\pi_{\mathrm{ref}}(y_w\mid x) + \log\pi_{\mathrm{ref}}(y_l\mid x) \]

reference model 冻结，所以对 \(\theta\) 求导时，reference 项不变。

8.1 只看与 \(\theta\) 有关的部分

\[ \Delta_\theta = \log\pi_\theta(y_w\mid x) - \log\pi_\theta(y_l\mid x) + \text{constant} \]

因此梯度方向直观上是：

\[ \text{提高 } \log\pi_\theta(y_w\mid x) \quad \text{降低 } \log\pi_\theta(y_l\mid x) \]

8.2 但提高和降低的强度由 sigmoid 控制

当 \(\Delta_\theta\) 很小甚至为负时：

\[ \sigma(\beta\Delta_\theta) \]

较小，loss 大，梯度强。模型会强烈纠正。

当 \(\Delta_\theta\) 已经很大时，sigmoid 接近 1，loss 接近 0，梯度变弱。

情况 含义 梯度强度
\(\Delta_\theta \lt 0\) 模型相对更偏向 rejected 强烈更新
\(\Delta_\theta \approx 0\) chosen 和 rejected 相对优势不明显 中等更新
\(\Delta_\theta \gg 0\) 模型已经明显偏向 chosen 更新变弱
一句话理解梯度： DPO 会提高 chosen 的 logprob，降低 rejected 的 logprob；但如果模型已经很好地区分二者，更新会自然变弱。

9. \(\beta\) 参数怎么理解？

DPO loss 中有一个重要参数：

\[ \beta \]

它来自 RLHF 目标中的 KL 系数。

9.1 在 RLHF 目标里

\[ \max_\pi \mathbb{E}[r] - \beta \mathrm{KL}(\pi\|\pi_{\mathrm{ref}}) \]

这里 \(\beta\) 越大，KL 惩罚越强，模型越不容易偏离 reference。

9.2 在 DPO loss 里

\[ L_{\mathrm{DPO}} = -\log\sigma(\beta\Delta_\theta) \]

这里 \(\beta\) 控制偏好差异的放大程度。

\(\beta\) 效果 可能风险
较小 更新更温和，偏好差距被压缩 学习慢，偏好优化弱
较大 偏好差距被放大，更新更激进 容易过拟合偏好数据，风格变化大
适中 保持偏好学习和稳定性平衡 需要实验调参
注意： 不同论文和框架中 \(\beta\) 的解释可能会和温度、KL 强度联系在一起。实践中不要只背“越大越强”这句话，要结合 loss 实现和训练现象观察。

10. DPO、SFT、RM、PPO 对比

方法 数据 目标 是否需要 Reward Model 是否需要 rollout
SFT prompt-response 模仿标准回答 不需要 不需要
Reward Model chosen / rejected 学习偏好评分函数 它本身就是 RM 不需要
RLHF-PPO prompt + 在线生成 response 最大化 RM reward 并控制 KL 需要 需要
DPO chosen / rejected 直接优化偏好概率 不显式需要 不需要

10.1 DPO 和 SFT 的区别

SFT 只看 chosen：

\[ L_{\mathrm{SFT}} = -\log\pi_\theta(y_w\mid x) \]

DPO 同时看 chosen 和 rejected：

\[ L_{\mathrm{DPO}} = -\log\sigma \left( \beta[ \text{chosen log ratio} - \text{rejected log ratio} ] \right) \]
核心差别： SFT 学“这个答案要生成出来”；DPO 学“这个答案要比另一个答案更值得生成”。

10.2 DPO 和 Reward Model 训练的区别

Reward Model loss 是：

\[ L_{\mathrm{RM}} = -\log\sigma(r_\phi(x,y_w)-r_\phi(x,y_l)) \]

DPO loss 是：

\[ L_{\mathrm{DPO}} = -\log\sigma \left( \beta[ \log\frac{\pi_\theta(y_w\mid x)}{\pi_{\mathrm{ref}}(y_w\mid x)} - \log\frac{\pi_\theta(y_l\mid x)}{\pi_{\mathrm{ref}}(y_l\mid x)} ] \right) \]

可以看到，DPO 把 reward difference 替换成了 policy log ratio difference。

10.3 DPO 和 PPO 的区别

维度 RLHF-PPO DPO
训练方式 online / on-policy 或近似 on-policy offline preference optimization
是否生成新回答 训练中需要 rollout 直接用偏好数据
是否训练 Value Head 通常需要 不需要
是否显式 Reward Model 需要 不需要
工程复杂度 高 低
探索能力 更强，可在线发现新样本 受限于已有偏好数据
一句话： DPO 更简单、更稳定、更像监督训练；PPO 更复杂，但可以在线探索和优化新的模型输出分布。

11. Sequence logprob 与 token logprob

DPO 公式里有：

\[ \log\pi_\theta(y\mid x) \]

这里 \(y\) 是整段回答，不是单个 token。

11.1 自回归展开

如果：

\[ y=(y_1,y_2,\ldots,y_T) \]

那么：

\[ \pi_\theta(y\mid x) = \prod_{t=1}^{T} \pi_\theta(y_t\mid x,y_{\lt t}) \]

取 log：

\[ \boxed{ \log\pi_\theta(y\mid x) = \sum_{t=1}^{T} \log\pi_\theta(y_t\mid x,y_{\lt t}) } \]
HTML 注意： 这里写 \(y_{\lt t}\)，不是直接写尖括号形式，避免浏览器把公式误解析成标签。

11.2 DPO 中 chosen 和 rejected 的 logprob

chosen 的序列 logprob：

\[ \log\pi_\theta(y_w\mid x) = \sum_{t=1}^{T_w} \log\pi_\theta(y_{w,t}\mid x,y_{w,\lt t}) \]

rejected 的序列 logprob：

\[ \log\pi_\theta(y_l\mid x) = \sum_{t=1}^{T_l} \log\pi_\theta(y_{l,t}\mid x,y_{l,\lt t}) \]

11.3 长度问题

因为 logprob 是 token logprob 的和，长回答通常会有更负的总 logprob。

做法 特点
sum logprob 符合序列概率定义，但受长度影响明显
average logprob 减弱长度影响，但不完全等于原始 DPO 推导
长度归一化 实践中可能有用，但要注意目标变化
实践坑： 如果数据里 chosen 普遍比 rejected 更长或更短，DPO 可能学到长度偏好，而不是真正质量偏好。

12. DPO 伪代码与最小实现逻辑

12.1 一条样本需要什么

sample = {
    "prompt": x,
    "chosen": y_w,
    "rejected": y_l
}

12.2 模型需要计算什么

policy_chosen_logp   = log πθ(y_w | x)
policy_rejected_logp = log πθ(y_l | x)

ref_chosen_logp      = log πref(y_w | x)
ref_rejected_logp    = log πref(y_l | x)

12.3 计算 log ratio difference

chosen_log_ratio = policy_chosen_logp - ref_chosen_logp
rejected_log_ratio = policy_rejected_logp - ref_rejected_logp

delta = chosen_log_ratio - rejected_log_ratio

12.4 计算 DPO loss

loss = - log_sigmoid(beta * delta)

12.5 伪代码完整流程

for batch in preference_dataset:

    prompt = batch["prompt"]
    chosen = batch["chosen"]
    rejected = batch["rejected"]

    # policy model logprob
    logp_chosen = policy.logprob(prompt, chosen)
    logp_rejected = policy.logprob(prompt, rejected)

    # reference model logprob, no grad
    with no_grad:
        ref_logp_chosen = ref_model.logprob(prompt, chosen)
        ref_logp_rejected = ref_model.logprob(prompt, rejected)

    chosen_log_ratio = logp_chosen - ref_logp_chosen
    rejected_log_ratio = logp_rejected - ref_logp_rejected

    logits = beta * (chosen_log_ratio - rejected_log_ratio)

    loss = -log_sigmoid(logits)

    loss.backward()
    optimizer.step()
实现重点： reference model 必须冻结；loss 只更新 policy model。

13. 常见误解与训练坑

13.1 误解一：DPO 就是 SFT 加 rejected 惩罚

不完全对。DPO 不是简单：

\[ -\log\pi_\theta(y_w\mid x) + \log\pi_\theta(y_l\mid x) \]

它有 reference model log ratio：

\[ \log \frac{\pi_\theta(y\mid x)} {\pi_{\mathrm{ref}}(y\mid x)} \]

这让它和 KL-constrained RLHF 有理论联系。

13.2 误解二：DPO 不需要 reference model

标准 DPO 需要 reference model。reference model 提供隐式 KL anchor。

如果去掉 reference model，训练目标会变得更像普通 pairwise ranking，失去 DPO 与 RLHF 目标之间的关键联系。

13.3 误解三：DPO 一定比 PPO 好

DPO 更简单，但不一定总是更强。

场景 DPO 表现 PPO / GRPO 可能优势
已有高质量偏好数据 很适合 不一定需要在线 RL
需要模型探索新解法 受限于数据 在线 RL 可以探索
数学 / 代码可验证任务 可用，但不是最自然 GRPO / RLVR 更直接
复杂 Agent 多轮环境 难覆盖轨迹空间 在线 RL 更适合

13.4 训练坑：偏好数据质量决定上限

DPO 是 offline preference optimization。它不会在训练中主动探索新回答，而是依赖已有 chosen / rejected 数据。

所以数据很关键： 如果 chosen 和 rejected 差异很浅，DPO 学到的东西也会很浅；如果偏好标注噪声大，DPO 会直接学习噪声。

13.5 训练坑：过度降低 rejected

某些 rejected 并不是完全错误，只是比 chosen 稍差。如果训练过猛，模型可能过度排斥某些合理表达。

13.6 训练坑：长度偏置

如果 chosen 总是更长，模型可能学会变长；如果 chosen 总是更短，模型可能学会过短回答。

这一节只做路线图，不展开推导。后续可以单独做 Stage 4.5。

方法 核心想法 适合什么时候看
DPO 直接用偏好对优化 policy log ratio 现在重点掌握
IPO 对偏好优化目标做不同形式的正则和回归处理 DPO 后进阶

15. 为什么下一阶段要学 GRPO / RLVR

DPO 解决的是偏好数据上的离线优化。

但是很多 reasoning 任务不只是“哪个回答更受人喜欢”，而是：

  • 数学答案是否正确；
  • 代码是否通过测试；
  • SQL 查询结果是否正确；
  • 工具调用参数是否正确；
  • Agent 任务是否完成。

这些任务有一个特点：

它们可以被自动验证。

所以 Stage 5 要进入： 

RLVR：Reinforcement Learning with Verifiable Reward
GRPO：Group Relative Policy Optimization

它们的核心问题是：

如果我们不需要人类偏好标注，而是可以用规则、单元测试、答案验证器直接给 reward，那么该怎样训练大模型？

16. 一页总结

概念 核心公式 你要理解什么
RLHF 目标 \(\max_\pi \mathbb{E}[r]-\beta KL(\pi\|\pi_{\mathrm{ref}})\) 高 reward 与不偏离 reference 的折中。
最优策略 \(\pi^*(y\mid x)=\frac{1}{Z(x)}\pi_{\mathrm{ref}}(y\mid x)\exp(r(x,y)/\beta)\) reference policy 被 reward 指数加权。
隐式奖励 \(r(x,y)=\beta\log\frac{\pi^*(y\mid x)}{\pi_{\mathrm{ref}}(y\mid x)}+\beta\log Z(x)\) reward 可以由最优策略和 reference 的 log ratio 表示。
偏好概率 \(\sigma(r_w-r_l)\) chosen 胜过 rejected 的概率。
DPO reward 差 \(\beta[\log\frac{\pi_\theta(y_w)}{\pi_{\mathrm{ref}}(y_w)}-\log\frac{\pi_\theta(y_l)}{\pi_{\mathrm{ref}}(y_l)}]\) 用 policy log ratio 差替代 reward 差。
DPO Loss \(-\log\sigma(\beta[\text{chosen log ratio}-\text{rejected log ratio}])\) 直接用偏好对优化 policy。
Reference Model \(\pi_{\mathrm{ref}}\) 提供隐式 KL anchor。
\(\beta\) 偏好差异缩放 / KL 相关参数 控制更新强度和稳定性。
进入 Stage 5 前的标准： 你应该能解释清楚：
  1. DPO 为什么不需要显式 Reward Model；
  2. DPO 为什么仍然需要 reference model；
  3. DPO loss 中 chosen log ratio 和 rejected log ratio 分别是什么；
  4. DPO 和 SFT 的区别；
  5. DPO 和 RLHF-PPO 的区别；
  6. 为什么数学、代码任务下一步更适合进入 GRPO / RLVR。

17. 本阶段练习

17.1 手推练习

  1. 从 \(\max_\pi \mathbb{E}[r]-\beta KL(\pi\|\pi_{\mathrm{ref}})\) 推导 \(\pi^*(y\mid x)\)。
  2. 从 \(\pi^*(y\mid x)\) 反推出 \(r(x,y)\)。
  3. 证明 \(r(x,y_w)-r(x,y_l)\) 中 \(\log Z(x)\) 会消掉。
  4. 把 Bradley-Terry 偏好模型与隐式 reward 结合，推导 DPO loss。
  5. 解释为什么 DPO 的目标不是简单的 \(-\log\pi(y_w)+\log\pi(y_l)\)。

17.2 编程练习

做一个最小 DPO loss 计算：

给定：
policy_chosen_logp = -10.0
policy_rejected_logp = -13.0
ref_chosen_logp = -11.0

计算：
