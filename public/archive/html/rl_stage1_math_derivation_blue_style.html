0. 本阶段学习目标

第一阶段只解决一个核心问题：

核心问题

一个智能体在不确定环境中行动，怎样用数学语言描述它的决策过程，并计算“某个状态或动作到底有多好”？

你学完这一阶段，应该能独立写出并解释以下公式：

\[ G_t = R_{t+1}+\gamma R_{t+2}+\gamma^2R_{t+3}+\cdots \] \[ v_\pi(s)=\mathbb{E}_\pi[G_t\mid S_t=s] \] \[ v_\pi(s)=\sum_a \pi(a\mid s)\sum_{s',r}p(s',r\mid s,a) \left[r+\gamma v_\pi(s')\right] \] \[ q_\pi(s,a)=\sum_{s',r}p(s',r\mid s,a) \left[r+\gamma \sum_{a'}\pi(a'\mid s')q_\pi(s',a')\right] \] \[ q_{k+1}(s,a)=q_k(s,a)+\alpha \left[r+\gamma\max_{a'}q_k(s',a')-q_k(s,a)\right] \]

先不要急着理解 PPO、DPO、GRPO。那些算法本质上都绕不开这几个问题：状态是什么，动作是什么，奖励是什么，价值是什么，策略怎样更新。

1. 记号系统：先统一语言

强化学习的公式看起来多，主要原因是它同时描述“时间”“随机性”“决策”和“奖励”。我们先把记号统一。

符号 含义 直观解释
\(S_t\) time step \(t\) 的状态 智能体当前看到的局面
\(A_t\) time step \(t\) 的动作 智能体当前做出的选择
\(R_{t+1}\) 执行 \(A_t\) 后获得的奖励 动作之后环境给出的分数
\(S_{t+1}\) 下一个状态 执行动作后进入的新局面
\(\pi(a\mid s)\) 策略 在状态 \(s\) 下选择动作 \(a\) 的概率
\(p(s',r\mid s,a)\) 环境转移概率 在 \(s\) 做 \(a\) 后，到 \(s'\) 并得到 \(r\) 的概率
\(\gamma\) 折扣因子 未来奖励的重要程度，通常 \(0\le \gamma \le 1\)
\(\alpha\) 学习率 每次更新时相信新信息的程度
小白最容易混淆的一点

\(R_{t+1}\) 不是 \(R_t\)。因为奖励通常发生在动作之后：在状态 \(S_t\) 选择动作 \(A_t\)，环境跳到 \(S_{t+1}\)，同时给你奖励 \(R_{t+1}\)。

因此一个最基本的交互链条是：

\[ S_t \xrightarrow{A_t} (R_{t+1},S_{t+1}) \]

2. Agent 与 Environment：先从交互过程开始

强化学习的对象不是单次预测，而是连续决策。智能体每一步都要根据当前状态选择动作，动作会影响未来状态，未来状态又影响之后的奖励。

第 1 步：环境给出状态

在时间 \(t\)，环境给智能体一个状态：

\[ S_t=s \]

例如走迷宫时，\(s\) 可以表示当前位置。玩游戏时，\(s\) 可以表示屏幕画面。大模型里，\(s\) 可以理解为 prompt 加上已经生成的上下文。

第 2 步：智能体根据策略选择动作

智能体不是随便选动作，而是根据策略 \(\pi\) 选动作：

\[ A_t \sim \pi(\cdot\mid S_t) \]

这句话的意思是：动作 \(A_t\) 是从条件概率分布 \(\pi(a\mid S_t)\) 中采样出来的。

第 3 步：环境根据转移概率返回新状态和奖励

智能体执行动作后，环境返回：

\[ (S_{t+1},R_{t+1})\sim p(\cdot,\cdot\mid S_t,A_t) \]

这里 \(p(s',r\mid s,a)\) 是环境本身的规律。智能体通常不知道它，只能通过和环境交互慢慢学习。

一句话理解强化学习

强化学习研究的是：当动作会影响未来时，智能体怎样通过奖励信号学会长期更优的行为策略。

3. MDP：强化学习的数学模型

MDP 全称是 Markov Decision Process，中文常译为马尔可夫决策过程。它是强化学习的基本数学模型。

3.1 MDP 的五元组

一个 MDP 通常写作：

\[ \mathcal{M}=(\mathcal{S},\mathcal{A},p,r,\gamma) \]
组成 含义
\(\mathcal{S}\) 状态空间，所有可能状态的集合
\(\mathcal{A}\) 动作空间，所有可能动作的集合
\(p(s',r\mid s,a)\) 转移概率，描述环境如何从当前状态动作转到下一个状态奖励
\(r\) 奖励函数，有时写成 \(r(s,a)\) 或 \(r(s,a,s')\)
\(\gamma\) 折扣因子，控制未来奖励的重要性

3.2 什么是 Markov 性质

Markov 性质的意思是：未来只依赖当前状态和当前动作，不需要依赖更早的历史。

严格写成：

\[ \Pr(S_{t+1}=s',R_{t+1}=r\mid S_0,A_0,R_1,\ldots,S_t=s,A_t=a) = \Pr(S_{t+1}=s',R_{t+1}=r\mid S_t=s,A_t=a) \]

这条公式很长，我们拆开看。

左边表示完整历史条件下的未来概率

\[ \Pr(S_{t+1}=s',R_{t+1}=r\mid S_0,A_0,R_1,\ldots,S_t=s,A_t=a) \]

它的意思是：如果我知道从最开始到现在的全部历史，那么下一步到 \(s'\) 并得到奖励 \(r\) 的概率是多少？

右边表示只知道当前状态动作时的未来概率

\[ \Pr(S_{t+1}=s',R_{t+1}=r\mid S_t=s,A_t=a) \]

它的意思是：如果我只知道现在状态是 \(s\)，当前动作是 \(a\)，下一步到 \(s'\) 并得到奖励 \(r\) 的概率是多少？

Markov 性质说这两者相等

也就是说，当前状态 \(S_t\) 已经压缩了所有对未来有用的信息。更早的历史不会额外改变未来概率。

为什么这个假设重要

如果没有 Markov 性质，我们就必须把全部历史都当成状态，数学会非常复杂。MDP 用“当前状态足够描述未来”这个假设，把连续决策问题变成可分析、可计算的问题。

4. 轨迹、概率与期望：为什么公式里总有 \(\mathbb{E}\)

强化学习中，智能体走出的整条路径叫 trajectory，中文可以叫轨迹。

\[ \tau=(S_0,A_0,R_1,S_1,A_1,R_2,S_2,\ldots) \]

4.1 一条轨迹的概率怎么写

假设初始状态分布是 \(\rho_0(s_0)\)，策略是 \(\pi(a\mid s)\)，环境转移是 \(p(s',r\mid s,a)\)。那么一条长度为 \(T\) 的轨迹概率为：

\[ \Pr(\tau) = \rho_0(s_0) \prod_{t=0}^{T-1} \pi(a_t\mid s_t) p(s_{t+1},r_{t+1}\mid s_t,a_t) \]

我们逐步推导这条公式。

先看初始状态

一条轨迹从 \(S_0=s_0\) 开始，它出现的概率是：

\[ \Pr(S_0=s_0)=\rho_0(s_0) \]

第 0 步选择动作

在状态 \(s_0\) 下选择动作 \(a_0\)，概率是：

\[ \Pr(A_0=a_0\mid S_0=s_0)=\pi(a_0\mid s_0) \]

环境转移到下一状态并给奖励

执行动作 \(a_0\) 后，环境给出 \(s_1,r_1\)，概率是：

\[ \Pr(S_1=s_1,R_1=r_1\mid S_0=s_0,A_0=a_0) = p(s_1,r_1\mid s_0,a_0) \]

一直乘下去

因为每一步都由“策略选择动作”和“环境返回状态奖励”组成，所以整条轨迹概率就是这些条件概率连乘。

4.2 为什么要取期望

同一个策略可能走出很多条轨迹。奖励也是随机的。因此我们不能只看某一次结果，而要看平均意义上的结果。这个平均就是期望。

\[ \mathbb{E}_\pi[\cdot] \]

下标 \(\pi\) 表示：这个期望是在策略 \(\pi\) 诱导出来的轨迹分布下计算的。

5. Return：为什么要折扣回报

强化学习关心的不是当前一步奖励，而是从当前时刻开始，未来所有奖励的总和。这个总和叫 return，记作 \(G_t\)。

5.1 不折扣的总回报

最直接的想法是把未来奖励全部加起来：

\[ G_t = R_{t+1}+R_{t+2}+R_{t+3}+\cdots \]

但如果任务很长，甚至无限长，这个和可能发散。例如每一步奖励都是 1：

\[ G_t=1+1+1+\cdots=\infty \]

5.2 引入折扣因子

为了让未来奖励逐渐变小，我们引入折扣因子 \(\gamma\)，得到：

\[ G_t = R_{t+1}+\gamma R_{t+2}+\gamma^2R_{t+3}+\gamma^3R_{t+4}+\cdots \]

其中：

\[ 0\le \gamma \le 1 \]

5.3 为什么这样可以防止发散

如果每一步奖励都是 1，则：

\[ G_t = 1+\gamma+\gamma^2+\gamma^3+\cdots \]

这是一个等比级数。当 \(0\le \gamma <1\) 时：

\[ 1+\gamma+\gamma^2+\gamma^3+\cdots = \frac{1}{1-\gamma} \]

所以只要 \(\gamma<1\)，无限长奖励也可以有有限总和。

5.4 Return 的递推形式

这是后面推导 Bellman 方程最关键的一步。我们从定义开始：

\[ G_t = R_{t+1}+\gamma R_{t+2}+\gamma^2R_{t+3}+\gamma^3R_{t+4}+\cdots \]

再写出下一时刻的回报：

\[ G_{t+1}=R_{t+2}+\gamma R_{t+3}+\gamma^2R_{t+4}+\cdots \]

把 \(G_{t+1}\) 乘上 \(\gamma\)：

\[ \gamma G_{t+1} = \gamma R_{t+2}+\gamma^2R_{t+3}+\gamma^3R_{t+4}+\cdots \]

与 \(G_t\) 对比：

\[ G_t = R_{t+1} + \underbrace{\gamma R_{t+2}+\gamma^2R_{t+3}+\gamma^3R_{t+4}+\cdots}_{\gamma G_{t+1}} \]

因此得到递推关系：

\[ \boxed{G_t=R_{t+1}+\gamma G_{t+1}} \]
这一步非常重要

Bellman 方程的本质就是把“从现在开始的长期价值”拆成“当前一步奖励”加上“下一状态的长期价值”。

6. Value Function：价值函数从哪里来

Return \(G_t\) 是某一次真实轨迹上的未来奖励总和。可是强化学习需要判断“一个状态通常好不好”，不能只看一次运气。因此要取期望。

6.1 状态价值函数

在策略 \(\pi\) 下，状态 \(s\) 的价值定义为：

\[ \boxed{ v_\pi(s)=\mathbb{E}_\pi[G_t\mid S_t=s] } \]

逐字解释：

部分 含义
\(v_\pi(s)\) 策略 \(\pi\) 下状态 \(s\) 的价值
\(\mathbb{E}_\pi\) 对策略 \(\pi\) 产生的所有可能轨迹取期望
\(G_t\) 从时间 \(t\) 开始的折扣回报
\(\mid S_t=s\) 条件是当前状态固定为 \(s\)

6.2 为什么写成条件期望

因为我们想问的是：如果现在已经站在状态 \(s\)，未来平均能得到多少奖励？

这不是无条件期望：

\[ \mathbb{E}_\pi[G_t] \]

而是条件期望：

\[ \mathbb{E}_\pi[G_t\mid S_t=s] \]

区别是：前者平均了所有可能起点，后者固定当前状态为 \(s\)。

6.3 动作价值函数

有时我们不只想知道状态好不好，还想知道在状态 \(s\) 下先做动作 \(a\) 好不好。于是定义动作价值函数：

\[ \boxed{ q_\pi(s,a)=\mathbb{E}_\pi[G_t\mid S_t=s,A_t=a] } \]

它问的是：如果当前状态是 \(s\)，并且第一步强制选择动作 \(a\)，之后继续按照策略 \(\pi\) 行动，那么未来平均回报是多少？

两个价值函数的区别

\(v_\pi(s)\) 评价状态，默认动作由策略 \(\pi\) 选择；\(q_\pi(s,a)\) 评价状态动作对，第一步动作固定为 \(a\)，之后再按 \(\pi\) 行动。

7. Bellman 期望方程：逐步推导

Bellman 方程是第一阶段最重要的公式。我们从状态价值函数的定义出发，一步一步推导。

7.1 从定义开始

\[ v_\pi(s)=\mathbb{E}_\pi[G_t\mid S_t=s] \]

7.2 用 Return 的递推形式替换 \(G_t\)

前面已经推导出：

\[ G_t=R_{t+1}+\gamma G_{t+1} \]

代入状态价值定义：

\[ v_\pi(s) = \mathbb{E}_\pi[R_{t+1}+\gamma G_{t+1}\mid S_t=s] \]

7.3 利用期望的线性性

期望有线性性质：

\[ \mathbb{E}[X+Y]=\mathbb{E}[X]+\mathbb{E}[Y] \]

所以：

\[ v_\pi(s) = \mathbb{E}_\pi[R_{t+1}\mid S_t=s] + \gamma \mathbb{E}_\pi[G_{t+1}\mid S_t=s] \]

但这样还不够，因为下一步状态 \(S_{t+1}\) 是随机的。我们需要把“下一步可能发生什么”展开。

7.4 对动作展开

当前状态为 \(s\) 时，策略 \(\pi\) 会以概率 \(\pi(a\mid s)\) 选择动作 \(a\)。所以我们对所有动作求和：

\[ v_\pi(s) = \sum_a \pi(a\mid s) \mathbb{E}_\pi[G_t\mid S_t=s,A_t=a] \]

这里用的是全期望公式：

\[ \mathbb{E}[X\mid S=s] = \sum_a \Pr(A=a\mid S=s)\mathbb{E}[X\mid S=s,A=a] \]

7.5 对环境转移展开

在状态 \(s\) 做动作 \(a\) 后，环境可能转移到不同的 \(s'\)，并给出不同奖励 \(r\)。其概率是：

\[ p(s',r\mid s,a) \]

所以继续展开：

\[ v_\pi(s) = \sum_a \pi(a\mid s) \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma \mathbb{E}_\pi[G_{t+1}\mid S_{t+1}=s'] \right] \]

7.6 识别下一状态的价值

根据状态价值函数定义：

\[ v_\pi(s')=\mathbb{E}_\pi[G_{t+1}\mid S_{t+1}=s'] \]

代回上式：

\[ \boxed{ v_\pi(s) = \sum_a \pi(a\mid s) \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma v_\pi(s') \right] } \]
Bellman 期望方程的直观含义

一个状态的价值，等于在这个状态下按策略选择动作后，所有可能下一步结果的“即时奖励 + 折扣后的下一状态价值”的加权平均。

7.7 用一句话压缩公式

\[ \text{当前状态价值} = \text{即时奖励的期望} + \gamma \times \text{下一状态价值的期望} \]

8. Q 函数：动作价值函数

只知道状态价值 \(v_\pi(s)\) 有时不够，因为决策时我们真正要比较的是：在同一个状态下，哪个动作更好。

8.1 Q 函数定义

\[ q_\pi(s,a) = \mathbb{E}_\pi[G_t\mid S_t=s,A_t=a] \]

8.2 \(v_\pi\) 和 \(q_\pi\) 的关系

状态价值是策略对所有动作的平均：

\[ v_\pi(s)=\sum_a \pi(a\mid s)q_\pi(s,a) \]

推导如下：

从 \(v_\pi(s)\) 定义开始

\[ v_\pi(s)=\mathbb{E}_\pi[G_t\mid S_t=s] \]

按动作 \(A_t\) 分类

\[ \mathbb{E}_\pi[G_t\mid S_t=s] = \sum_a \Pr(A_t=a\mid S_t=s) \mathbb{E}_\pi[G_t\mid S_t=s,A_t=a] \]

代入策略与 Q 函数定义

\[ \Pr(A_t=a\mid S_t=s)=\pi(a\mid s) \] \[ \mathbb{E}_\pi[G_t\mid S_t=s,A_t=a]=q_\pi(s,a) \]

所以：

\[ \boxed{ v_\pi(s)=\sum_a \pi(a\mid s)q_\pi(s,a) } \]

8.3 从 Q 函数反推策略

如果我们知道每个动作的 \(q(s,a)\)，最自然的决策就是选最大 Q 的动作：

\[ \pi(s)=\arg\max_a q(s,a) \]

这也是后面 Q-learning 能做控制的原因：只要学会 Q 函数，就可以直接选动作。

9. Bellman 方程的 Q 形式

现在我们推导 \(q_\pi(s,a)\) 的 Bellman 方程。

9.1 从 Q 函数定义开始

\[ q_\pi(s,a)=\mathbb{E}_\pi[G_t\mid S_t=s,A_t=a] \]

9.2 代入 Return 递推

\[ G_t=R_{t+1}+\gamma G_{t+1} \]

因此：

\[ q_\pi(s,a) = \mathbb{E}_\pi[R_{t+1}+\gamma G_{t+1}\mid S_t=s,A_t=a] \]

9.3 对环境的下一状态和奖励展开

因为当前 \(s,a\) 已经固定，下一步随机性来自环境：

\[ q_\pi(s,a) = \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma \mathbb{E}_\pi[G_{t+1}\mid S_{t+1}=s'] \right] \]

9.4 下一状态后还要按策略选择动作

到达 \(s'\) 后，智能体会按照策略 \(\pi(a'\mid s')\) 选择下一动作 \(a'\)。所以：

\[ \mathbb{E}_\pi[G_{t+1}\mid S_{t+1}=s'] = v_\pi(s') = \sum_{a'}\pi(a'\mid s')q_\pi(s',a') \]

9.5 代入得到 Q 的 Bellman 期望方程

\[ \boxed{ q_\pi(s,a) = \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma \sum_{a'}\pi(a'\mid s')q_\pi(s',a') \right] } \]
注意

\(v_\pi(s)\) 的 Bellman 方程先对当前动作 \(a\) 求平均；\(q_\pi(s,a)\) 的 Bellman 方程当前动作已经固定，所以先对环境转移求平均，然后再对下一状态的动作 \(a'\) 求平均。

10. 最优价值与 Bellman 最优方程

前面的 \(v_\pi\) 和 \(q_\pi\) 都依赖某个给定策略 \(\pi\)。但强化学习的目标通常不是评价一个固定策略，而是找到最优策略。

10.1 最优状态价值函数

\[ v_*(s)=\max_\pi v_\pi(s) \]

它表示：在所有可能策略里，从状态 \(s\) 出发能获得的最大期望回报。

10.2 最优动作价值函数

\[ q_*(s,a)=\max_\pi q_\pi(s,a) \]

它表示：在状态 \(s\) 先做动作 \(a\)，之后采用最优策略时，能得到的最大期望回报。

10.3 最优策略与 \(q_*\)

如果知道 \(q_*(s,a)\)，最优动作直接由最大值给出：

\[ \pi_*(s)=\arg\max_a q_*(s,a) \]

10.4 Bellman 最优方程：状态价值形式

由于最优策略会选择使未来价值最大的动作，所以：

\[ \boxed{ v_*(s) = \max_a \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma v_*(s') \right] } \]

和 Bellman 期望方程相比，最大的变化是：

\[ \sum_a \pi(a\mid s)(\cdot) \quad\Longrightarrow\quad \max_a(\cdot) \]

也就是说，不再按照某个策略加权平均，而是直接选择最好的动作。

10.5 Bellman 最优方程：动作价值形式

对 \(q_*(s,a)\)，当前动作 \(a\) 已经固定。执行它后到达 \(s'\)，下一步开始选择最优动作：

\[ \boxed{ q_*(s,a) = \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma\max_{a'}q_*(s',a') \right] } \]
最重要的一句话

Q-learning 的更新公式，本质上就是在用样本近似这个 \(q_*\) 的 Bellman 最优方程。

11. Policy Evaluation：策略评估

策略评估的任务是：给定一个策略 \(\pi\)，计算它的价值函数 \(v_\pi\)。

11.1 从 Bellman 期望方程出发

\[ v_\pi(s) = \sum_a \pi(a\mid s) \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma v_\pi(s') \right] \]

11.2 把它看成一个方程组

如果状态空间有限，比如：

\[ \mathcal{S}=\{s_1,s_2,\ldots,s_n\} \]

那么每个状态都有一个方程：

\[ v_\pi(s_1)=\cdots \] \[ v_\pi(s_2)=\cdots \] \[ \vdots \] \[ v_\pi(s_n)=\cdots \]

未知数就是：

\[ v_\pi(s_1),v_\pi(s_2),\ldots,v_\pi(s_n) \]

11.3 迭代求解

直接解线性方程组有时很贵。强化学习常用迭代形式：

\[ v_{k+1}(s) = \sum_a \pi(a\mid s) \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma v_k(s') \right] \]

其中 \(v_k\) 是第 \(k\) 次估计，\(v_{k+1}\) 是下一次估计。

11.4 为什么这个迭代合理

Bellman 方程要求：

\[ v = T_\pi v \]

其中 \(T_\pi\) 是 Bellman 期望算子：

\[ (T_\pi v)(s) = \sum_a \pi(a\mid s) \sum_{s',r}p(s',r\mid s,a) \left[r+\gamma v(s')\right] \]

迭代就是不断做：

\[ v_{k+1}=T_\pi v_k \]

当 \(0\le \gamma<1\) 且状态动作有限时，这个迭代会逐渐收敛到 \(v_\pi\)。

12. Policy Improvement：策略改进

策略评估告诉我们当前策略有多好。策略改进要回答：怎样基于当前价值函数得到更好的策略？

12.1 用 \(q_\pi(s,a)\) 比较动作

在状态 \(s\) 下，如果某个动作 \(a\) 满足：

\[ q_\pi(s,a)>v_\pi(s) \]

这说明：先做动作 \(a\)，再按照原策略 \(\pi\) 走，比直接按原策略平均选择动作更好。

12.2 贪心改进

于是定义新策略：

\[ \pi'(s)=\arg\max_a q_\pi(s,a) \]

这叫 greedy policy improvement，贪心策略改进。

12.3 策略迭代

策略迭代就是反复做两步：

\[ \pi_0 \xrightarrow{\text{policy evaluation}} v_{\pi_0} \xrightarrow{\text{policy improvement}} \pi_1 \xrightarrow{\text{policy evaluation}} v_{\pi_1} \xrightarrow{\text{policy improvement}} \pi_2 \rightarrow \cdots \]

直到策略不再变化，就得到最优策略。

策略迭代的思想

先评价当前策略，再根据评价结果改进策略。这个“评价—改进”的循环，是许多强化学习算法的基本骨架。

13. Value Iteration：价值迭代

价值迭代可以看成把策略评估和策略改进合并在一起。它直接使用 Bellman 最优方程。

13.1 Bellman 最优方程

\[ v_*(s) = \max_a \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma v_*(s') \right] \]

13.2 迭代形式

用当前估计 \(v_k\) 替代未知的 \(v_*\)，得到更新：

\[ \boxed{ v_{k+1}(s) = \max_a \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma v_k(s') \right] } \]

13.3 价值迭代的算法流程

初始化：对所有状态 s，设置 V(s)=0

重复直到收敛：
    对每个状态 s：
        对每个动作 a：
            计算执行 a 后的期望价值
            value(a) = Σ_{s',r} p(s',r|s,a)[r + γV(s')]
        更新：
            V(s) = max_a value(a)

最后得到策略：
    π(s) = argmax_a Σ_{s',r} p(s',r|s,a)[r + γV(s')]

13.4 一个小例子

假设某状态 \(s\) 有两个动作 \(a_1,a_2\)。动作 \(a_1\) 后必定到 \(s_1\)，奖励 1；动作 \(a_2\) 后必定到 \(s_2\)，奖励 0。设 \(\gamma=0.9\)，当前估计：

\[ v_k(s_1)=10,\qquad v_k(s_2)=15 \]

那么动作 \(a_1\) 的价值估计为：

\[ 1+0.9\times 10=10 \]

动作 \(a_2\) 的价值估计为：

\[ 0+0.9\times 15=13.5 \]

取最大：

\[ v_{k+1}(s)=\max(10,13.5)=13.5 \]

虽然 \(a_2\) 眼前奖励为 0，但它通往更有价值的未来，所以更优。

14. Monte Carlo 与 TD：不知道环境模型怎么办

前面的 policy evaluation 和 value iteration 都用了 \(p(s',r\mid s,a)\)。但现实中我们往往不知道环境转移概率。

这时我们不再用完整期望，而是用样本估计。

14.1 Monte Carlo 方法

Monte Carlo 的思想是：等一整条轨迹结束后，直接用真实回报 \(G_t\) 作为目标。

\[ V(S_t)\leftarrow V(S_t)+\alpha\left[G_t-V(S_t)\right] \]

这条公式来自均值更新。假设当前估计是 \(V(S_t)\)，新观察到的样本目标是 \(G_t\)，那么：

\[ \text{新估计} = \text{旧估计} + \alpha(\text{目标}-\text{旧估计}) \]

14.2 TD 方法

TD 全称 Temporal Difference，中文常译为时序差分。它不等整条轨迹结束，而是只看一步：

\[ G_t=R_{t+1}+\gamma G_{t+1} \]

但是 \(G_{t+1}\) 不知道，于是用当前估计 \(V(S_{t+1})\) 近似：

\[ G_t \approx R_{t+1}+\gamma V(S_{t+1}) \]

于是 TD 目标是：

\[ \text{TD target}=R_{t+1}+\gamma V(S_{t+1}) \]

TD 误差是：

\[ \delta_t=R_{t+1}+\gamma V(S_{t+1})-V(S_t) \]

更新公式是：

\[ \boxed{ V(S_t)\leftarrow V(S_t)+\alpha\delta_t } \]

即：

\[ \boxed{ V(S_t)\leftarrow V(S_t)+\alpha \left[ R_{t+1}+\gamma V(S_{t+1})-V(S_t) \right] } \]

14.3 MC 与 TD 的区别

方法 目标 是否等轨迹结束 特点
Monte Carlo \(G_t\) 需要 目标真实，但方差大
TD \(R_{t+1}+\gamma V(S_{t+1})\) 不需要 更新快，但目标使用了估计

15. Q-learning 从 Bellman 最优方程推导

Q-learning 是第一个你必须真正吃透的控制算法。它的核心是学习最优动作价值函数 \(q_*(s,a)\)。

15.1 从 Bellman 最优方程开始

\[ q_*(s,a) = \sum_{s',r} p(s',r\mid s,a) \left[ r+\gamma\max_{a'}q_*(s',a') \right] \]

15.2 现实中没有完整转移概率

如果不知道 \(p(s',r\mid s,a)\)，就没法对所有 \(s',r\) 求和。我们只能从环境中采样一条经验：

\[ (s,a,r,s') \]

这表示：在状态 \(s\) 做动作 \(a\)，得到奖励 \(r\)，进入下一状态 \(s'\)。

15.3 用单个样本近似期望

Bellman 最优方程右边的期望目标：

\[ \mathbb{E}\left[ R_{t+1}+\gamma\max_{a'}q_*(S_{t+1},a') \mid S_t=s,A_t=a \right] \]

用一次采样近似为：

\[ r+\gamma\max_{a'}Q(s',a') \]

15.4 构造 TD 误差

当前估计是：

\[ Q(s,a) \]

新目标是：

\[ r+\gamma\max_{a'}Q(s',a') \]

所以误差是：

\[ \delta = r+\gamma\max_{a'}Q(s',a')-Q(s,a) \]

15.5 得到 Q-learning 更新公式

用“旧估计 + 学习率 × 误差”的形式更新：

\[ Q(s,a)\leftarrow Q(s,a)+\alpha\delta \]

代入 \(\delta\)：

\[ \boxed{ Q(s,a) \leftarrow Q(s,a) + \alpha \left[ r+\gamma\max_{a'}Q(s',a')-Q(s,a) \right] } \]

15.6 为什么叫 off-policy

Q-learning 更新目标中使用的是：

\[ \max_{a'}Q(s',a') \]

这表示它假设下一步采用贪心最优动作。但实际采样时，智能体可能为了探索而选择了随机动作。例如用 \(\epsilon\)-greedy：

\[ A_t= \begin{cases} \text{随机动作}, & \text{概率 } \epsilon \\ \arg\max_a Q(S_t,a), & \text{概率 } 1-\epsilon \end{cases} \]

行为策略用于探索，更新目标却朝最优贪心策略学习，因此称为 off-policy。

15.7 Q-learning 伪代码

初始化 Q(s,a)=0

for episode in episodes:
    初始化状态 s
    while s 不是终止状态:
        用 ε-greedy 根据 Q 选择动作 a
        执行动作 a，得到奖励 r 和下一状态 s'
        target = r + γ * max_{a'} Q(s', a')
        error = target - Q(s, a)
        Q(s, a) = Q(s, a) + α * error
        s = s'
小白要特别记住

Q-learning 的公式不是凭空来的。它来自 Bellman 最优方程，只是把完整期望换成了一次环境采样，把未知的 \(q_*\) 换成当前估计 \(Q\)。

16. 和大模型强化学习的对应关系

现在把第一阶段的数学对象映射到大模型里。后面你学 RLHF、PPO、DPO、GRPO 时会反复用到这些对应关系。

传统 RL 大模型后训练中的对应物 解释
状态 \(S_t\) prompt + 已生成 token 模型当前看到的上下文
动作 \(A_t\) 下一个 token，或一整段 response 语言模型的输出选择
策略 \(\pi(a\mid s)\) 语言模型的 token 概率分布 模型在上下文下生成各 token 的概率
奖励 \(R\) 人类偏好、规则奖励、答案正确性、测试通过率 评价回答好坏的信号
价值函数 \(V\) Value Head / Critic PPO 中估计当前生成轨迹的未来收益
Q 函数 \(Q(s,a)\) 某个 token 或动作选择后的长期收益 在 LLM 中显式 Q-learning 不常见，但思想存在
轨迹 \(\tau\) 完整生成过程 / 多轮工具调用过程 从 prompt 到最终答案的行为序列
Return \(G_t\) 最终奖励或过程奖励的折扣累计 回答质量、推理正确性、工具调用成功率等
关键理解

大模型不是“套了一层强化学习名词”。语言模型本身就是一个策略分布：给定上下文 \(s\)，输出下一个 token \(a\) 的概率 \(\pi_\theta(a\mid s)\)。后训练中的 RL，就是在奖励信号下调整这个策略分布。

16.1 为什么这个阶段对 PPO 很重要

PPO 本质上仍然在优化策略：

\[ \pi_\theta(a\mid s) \]

只是它会限制新策略不要离旧策略太远。后面你会看到 KL 惩罚：

\[ \mathrm{KL}(\pi_\theta \,\|\, \pi_{\mathrm{ref}}) \]

这可以理解为：模型可以为了更高奖励而改变输出分布，但不能改得太离谱。

18. 一页总结

第一阶段你只需要抓住一条主线：

\[ \text{交互} \Rightarrow \text{轨迹} \Rightarrow \text{回报} \Rightarrow \text{价值函数} \Rightarrow \text{Bellman 方程} \Rightarrow \text{动态规划 / TD / Q-learning} \]
核心公式 你要理解的含义
\(G_t=R_{t+1}+\gamma G_{t+1}\) 长期回报可以拆成当前奖励和未来回报
\(v_\pi(s)=\mathbb{E}_\pi[G_t\mid S_t=s]\) 状态价值是从该状态出发的期望长期回报
\(q_\pi(s,a)=\mathbb{E}_\pi[G_t\mid S_t=s,A_t=a]\) 动作价值是先做某动作后的期望长期回报
\(v_\pi(s)=\sum_a\pi(a\mid s)\sum_{s',r}p(s',r\mid s,a)[r+\gamma v_\pi(s')]\) Bellman 期望方程，评价固定策略
\(q_*(s,a)=\sum_{s',r}p(s',r\mid s,a)[r+\gamma\max_{a'}q_*(s',a')]\) Bellman 最优方程，寻找最优动作价值
\(Q(s,a)\leftarrow Q(s,a)+\alpha[r+\gamma\max_{a'}Q(s',a')-Q(s,a)]\) Q-learning 用样本近似 Bellman 最优方程
进入第二阶段前的标准

你不需要把所有公式背下来，但必须能从 Return 的递推式推导出 Bellman 方程，并能解释 Q-learning 的 target 为什么是 \(r+\gamma\max_{a'}Q(s',a')\)。达到这个标准，再学 Policy Gradient 和 PPO 就不会漂。
