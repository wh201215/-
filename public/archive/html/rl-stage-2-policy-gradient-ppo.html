强化学习大模型路线｜Stage 2：Policy Gradient / Actor-Critic / GAE / PPO 数学推导

学习定位：Stage 1 你已经理解了 MDP、Return、Value Function、Bellman Equation 和 Q-learning。Stage 2 开始进入大模型后训练真正相关的部分：策略梯度 Policy Gradient。

为什么这一阶段重要？因为大模型本身就是一个策略：

\[ \pi_\theta(a\mid s) \]

给定上下文 \(s\)，模型输出下一个 token \(a\) 的概率。RLHF、PPO、GRPO 本质上都在调整这个策略分布。

Stage 2 目标：

  1. 理解为什么 Q-learning 不适合直接作为大模型后训练主线；
  2. 推导 Policy Gradient 的核心公式；
  3. 理解 REINFORCE 为什么可以用采样轨迹更新策略；
  4. 理解 baseline、advantage、Actor-Critic、GAE；
  5. 推导 PPO 的 clip objective，为后面 RLHF 做准备。

1. 为什么从 Q-learning 转向 Policy Gradient？

Stage 1 里，Q-learning 的核心是学习：

\[ Q(s,a) \]

然后用：

\[ a^*=\arg\max_a Q(s,a) \]

选择动作。

这在小动作空间里很好用。例如 GridWorld 里动作只有： 

up, down, left, right

但是到了大模型，动作空间非常巨大。

关键问题： 语言模型每一步的动作是“选择下一个 token”。如果词表有 50,000 个 token，那么每一步动作空间就是 50,000。
任务 动作空间 Q-learning 是否方便
GridWorld 4 个动作 方便
Atari 游戏 几十个动作 还可以
机器人连续控制 连续动作 不方便
大模型生成 几万 token，每步一个 token 非常不方便

所以大模型后训练更自然的方式不是显式学习每个动作的 \(Q(s,a)\)，而是直接学习策略：

\[ \pi_\theta(a\mid s) \]

也就是让模型直接输出动作概率分布。

一句话理解： Q-learning 是先学“每个动作有多好”，再选最好的动作；Policy Gradient 是直接调整“好动作出现的概率”。

2. 参数化策略 \(\pi_\theta(a\mid s)\)

Policy Gradient 的出发点是：策略不是一个固定表格，而是一个带参数的函数。

\[ \pi_\theta(a\mid s) \]

其中：

符号 含义 大模型中对应什么
\(\theta\) 策略参数 神经网络参数，也就是 LLM 的权重
\(s\) 状态 prompt + 已生成上下文
\(a\) 动作 下一个 token，或者一整段 response
\(\pi_\theta(a\mid s)\) 动作概率 模型在上下文中生成某个 token 的概率

2.1 离散动作策略

如果动作是离散的，例如： 

a ∈ {up, down, left, right}

策略可以输出一个 softmax 分布：

\[ \pi_\theta(a\mid s) = \frac{\exp(f_\theta(s,a))} {\sum_{a'}\exp(f_\theta(s,a'))} \]

这里 \(f_\theta(s,a)\) 是神经网络给动作 \(a\) 的打分，也叫 logit。

2.2 大模型中的 token 策略

大模型也是一样。给定上下文 \(s\)，模型输出每个 token 的 logit：

\[ z_\theta(s)=(z_1,z_2,\ldots,z_{|\mathcal{V}|}) \]

其中 \(|\mathcal{V}|\) 是词表大小。经过 softmax 得到每个 token 的概率：

\[ \pi_\theta(a_i\mid s) = \frac{\exp(z_i)} {\sum_{j=1}^{|\mathcal{V}|}\exp(z_j)} \]
大模型视角： LLM 的 next-token prediction 本身就是一个策略分布。后训练的 RL 不是把模型变成策略，而是对已有策略进行奖励驱动的优化。

3. 强化学习目标函数 \(J(\theta)\)

既然策略有参数 \(\theta\)，我们就要定义一个目标函数，然后让 \(\theta\) 朝着目标变好的方向更新。

3.1 目标：最大化期望回报

最自然的目标是：

\[ J(\theta)=\mathbb{E}_{\tau\sim p_\theta(\tau)}[G(\tau)] \]

逐个解释：

符号 含义
\(\tau\) 一条轨迹，例如 \(s_0,a_0,r_1,s_1,a_1,r_2,\ldots\)
\(p_\theta(\tau)\) 策略 \(\pi_\theta\) 诱导出的轨迹概率
\(G(\tau)\) 这条轨迹的总回报
\(J(\theta)\) 策略参数为 \(\theta\) 时的平均表现
一句话理解： \(J(\theta)\) 表示当前策略平均能拿多少奖励。Policy Gradient 的目标就是调整 \(\theta\)，让 \(J(\theta)\) 变大。

3.2 轨迹概率展开

根据 Stage 1 的内容，一条轨迹概率可以写成：

\[ p_\theta(\tau) = \rho_0(s_0) \prod_{t=0}^{T-1} \pi_\theta(a_t\mid s_t) p(s_{t+1},r_{t+1}\mid s_t,a_t) \]

注意，这里只有策略项 \(\pi_\theta(a_t\mid s_t)\) 含有参数 \(\theta\)。环境转移概率：

\[ p(s_{t+1},r_{t+1}\mid s_t,a_t) \]

不由智能体控制，所以通常不对 \(\theta\) 求导。

3.3 我们真正想求什么

为了最大化 \(J(\theta)\)，我们要计算梯度：

\[ \nabla_\theta J(\theta) \]

然后做梯度上升：

\[ \theta \leftarrow \theta+\alpha\nabla_\theta J(\theta) \]

注意这里是加号，因为我们要最大化奖励。如果写成 loss 最小化，才会用减号。

4. log-derivative trick

Policy Gradient 推导最关键的技巧叫：

log-derivative trick，也叫 score function trick。

它的核心公式是：

\[ \nabla_\theta p_\theta(x) = p_\theta(x)\nabla_\theta \log p_\theta(x) \]

这一步非常重要，我们慢慢推。

4.1 从对数求导开始

根据链式法则：

\[ \nabla_\theta \log p_\theta(x) = \frac{1}{p_\theta(x)} \nabla_\theta p_\theta(x) \]

两边同时乘上 \(p_\theta(x)\)：

\[ p_\theta(x)\nabla_\theta \log p_\theta(x) = \nabla_\theta p_\theta(x) \]

所以得到：

\[ \boxed{ \nabla_\theta p_\theta(x) = p_\theta(x)\nabla_\theta \log p_\theta(x) } \]

4.2 为什么这个技巧重要？

因为我们要求的是：

\[ \nabla_\theta J(\theta) = \nabla_\theta \sum_\tau p_\theta(\tau)G(\tau) \]

难点是 \(p_\theta(\tau)\) 是一个复杂轨迹概率。用 log-derivative trick 后，可以把：

\[ \nabla_\theta p_\theta(\tau) \]

变成：

\[ p_\theta(\tau)\nabla_\theta \log p_\theta(\tau) \]

这样就可以转回期望形式，用采样轨迹估计。

5. Policy Gradient 定理逐步推导

现在开始推导最核心公式。

5.1 从目标函数开始

\[ J(\theta)=\mathbb{E}_{\tau\sim p_\theta(\tau)}[G(\tau)] \]

如果轨迹空间是离散的，可以写成求和：

\[ J(\theta)=\sum_\tau p_\theta(\tau)G(\tau) \]

5.2 对 \(\theta\) 求梯度

\[ \nabla_\theta J(\theta) = \nabla_\theta \sum_\tau p_\theta(\tau)G(\tau) \]

因为 \(G(\tau)\) 是这条轨迹的回报，先把它看成对 \(\theta\) 不直接求导的样本函数，于是：

\[ \nabla_\theta J(\theta) = \sum_\tau \nabla_\theta p_\theta(\tau) G(\tau) \]

5.3 使用 log-derivative trick

用：

\[ \nabla_\theta p_\theta(\tau) = p_\theta(\tau)\nabla_\theta \log p_\theta(\tau) \]

代入：

\[ \nabla_\theta J(\theta) = \sum_\tau p_\theta(\tau) \nabla_\theta \log p_\theta(\tau) G(\tau) \]

5.4 重新写成期望

因为：

\[ \sum_\tau p_\theta(\tau)f(\tau) = \mathbb{E}_{\tau\sim p_\theta(\tau)}[f(\tau)] \]

所以：

\[ \nabla_\theta J(\theta) = \mathbb{E}_{\tau\sim p_\theta(\tau)} \left[ \nabla_\theta \log p_\theta(\tau)G(\tau) \right] \]

5.5 展开轨迹 log 概率

轨迹概率为：

\[ p_\theta(\tau) = \rho_0(s_0) \prod_{t=0}^{T-1} \pi_\theta(a_t\mid s_t) p(s_{t+1},r_{t+1}\mid s_t,a_t) \]

取 log：

\[ \log p_\theta(\tau) = \log \rho_0(s_0) + \sum_{t=0}^{T-1}\log \pi_\theta(a_t\mid s_t) + \sum_{t=0}^{T-1} \log p(s_{t+1},r_{t+1}\mid s_t,a_t) \]

对 \(\theta\) 求梯度。初始分布和环境转移概率不含 \(\theta\)，梯度为 0：

\[ \nabla_\theta \log p_\theta(\tau) = \sum_{t=0}^{T-1} \nabla_\theta \log \pi_\theta(a_t\mid s_t) \]

5.6 得到基础 Policy Gradient 公式

代回前面：

\[ \boxed{ \nabla_\theta J(\theta) = \mathbb{E}_{\tau\sim p_\theta(\tau)} \left[ \sum_{t=0}^{T-1} \nabla_\theta \log \pi_\theta(a_t\mid s_t) G(\tau) \right] } \]

这就是最基础的轨迹级 Policy Gradient。

直观理解： 如果一条轨迹回报 \(G(\tau)\) 很高，就提高这条轨迹中所有动作的概率；如果回报低，就降低这些动作的概率。

5.7 从整条轨迹回报改成从当前时刻开始的回报

更常见的写法是：

\[ \boxed{ \nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta} \left[ \sum_{t=0}^{T-1} \nabla_\theta \log \pi_\theta(A_t\mid S_t) G_t \right] } \]

这里的 \(G_t\) 是从第 \(t\) 步开始的回报：

\[ G_t=R_{t+1}+\gamma R_{t+2}+\gamma^2R_{t+3}+\cdots \]

为什么可以用 \(G_t\) 而不是整条 \(G(\tau)\)？因为动作 \(A_t\) 不可能影响过去已经发生的奖励，只能影响当前及未来的奖励。

重要理解： 在第 \(t\) 步选择动作时，过去奖励已经发生，和当前动作没有因果关系。所以给第 \(t\) 步动作分配信用时，应该使用从 \(t\) 开始的未来回报 \(G_t\)。

6. REINFORCE 算法

REINFORCE 是最基础的 Monte Carlo Policy Gradient 算法。

6.1 REINFORCE 更新公式

\[ \theta \leftarrow \theta + \alpha G_t \nabla_\theta \log \pi_\theta(A_t\mid S_t) \]

如果一条 episode 有多个时间步，就对每一步都更新：

\[ \theta \leftarrow \theta + \alpha \sum_{t=0}^{T-1} G_t \nabla_\theta \log \pi_\theta(A_t\mid S_t) \]

6.2 这个公式到底在干什么？

部分 作用
\(\nabla_\theta \log \pi_\theta(A_t\mid S_t)\) 告诉参数如何变化才能提高当前动作概率。
\(G_t\) 告诉这个动作后面的结果好不好。
\(\alpha\) 学习率，控制更新幅度。
一句话： 如果 \(G_t\) 大，说明这个动作之后表现好，就提高这个动作概率；如果 \(G_t\) 小或为负，就降低这个动作概率。

6.3 REINFORCE 伪代码

初始化策略参数 θ

for each episode:
    用当前策略 πθ 采样一条完整轨迹:
        S0, A0, R1, S1, A1, R2, ...

    对每个时间步 t:
        计算从 t 开始的回报 Gt
        更新参数:
        θ ← θ + α Gt ∇θ log πθ(At | St)

6.4 REINFORCE 的问题

REINFORCE 虽然简单，但方差很大。

为什么方差大？ 同一个动作有时因为后续运气好得到高回报，有时因为环境随机性得到低回报。直接用整段回报 \(G_t\) 更新，会导致梯度非常不稳定。

解决思路是加入 baseline。

7. Baseline 为什么不改变期望梯度？

我们可以把更新中的 \(G_t\) 换成：

\[ G_t-b(S_t) \]

其中 \(b(S_t)\) 是 baseline，通常取状态价值函数：

\[ b(S_t)=V^\pi(S_t) \]

7.1 加 baseline 后的梯度

\[ \nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta} \left[ \nabla_\theta \log \pi_\theta(A_t\mid S_t) \left(G_t-b(S_t)\right) \right] \]

关键问题是：为什么减去 \(b(S_t)\) 不会改变期望梯度？

7.2 证明 baseline 项期望为 0

我们只看 baseline 那一项：

\[ \mathbb{E}_{A_t\sim \pi_\theta} \left[ \nabla_\theta \log \pi_\theta(A_t\mid S_t)b(S_t) \right] \]

在给定 \(S_t=s\) 时，\(b(s)\) 与动作 \(a\) 无关，可以提出：

\[ b(s) \sum_a \pi_\theta(a\mid s) \nabla_\theta \log \pi_\theta(a\mid s) \]

使用 log-derivative trick：

\[ \pi_\theta(a\mid s) \nabla_\theta \log \pi_\theta(a\mid s) = \nabla_\theta \pi_\theta(a\mid s) \]

所以：

\[ b(s) \sum_a \nabla_\theta \pi_\theta(a\mid s) = b(s) \nabla_\theta \sum_a \pi_\theta(a\mid s) \]

由于策略概率和为 1：

\[ \sum_a \pi_\theta(a\mid s)=1 \]

所以：

\[ b(s)\nabla_\theta 1=0 \]

因此 baseline 项的期望为 0。

结论： 减去只依赖状态、不依赖动作的 baseline，不会改变梯度期望，但可以降低方差。

8. Advantage Function

当 baseline 取状态价值函数：

\[ b(S_t)=V^\pi(S_t) \]

就得到 Advantage：

\[ A^\pi(S_t,A_t)=Q^\pi(S_t,A_t)-V^\pi(S_t) \]

8.1 Advantage 的直观含义

Advantage 表示： 某个动作比当前状态下的平均动作好多少。

如果：

\[ A^\pi(s,a)>0 \]

说明动作 \(a\) 比平均水平好，应该提高概率。

如果：

\[ A^\pi(s,a)<0 \]

说明动作 \(a\) 比平均水平差，应该降低概率。

8.2 Policy Gradient 的 Advantage 形式

\[ \boxed{ \nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta} \left[ \nabla_\theta \log \pi_\theta(A_t\mid S_t) A^\pi(S_t,A_t) \right] } \]

这是后面 PPO、GRPO 都会用到的核心结构。

大模型后训练里的理解： 不是所有高概率 token 都应该继续提高概率，而是那些“比当前模型平均表现更好”的生成动作应该提高概率。

9. Actor-Critic

Actor-Critic 是 Policy Gradient 的重要改进。

组件 作用 对应函数
Actor 负责选择动作 \(\pi_\theta(a\mid s)\)
Critic 负责评价状态或动作 \(V_\phi(s)\) 或 \(Q_\phi(s,a)\)

9.1 为什么需要 Critic？

REINFORCE 用完整回报 \(G_t\) 估计动作好坏，方差很大。Critic 用神经网络估计价值函数：

\[ V_\phi(s)\approx V^\pi(s) \]

然后用它作为 baseline 或 advantage 估计。

9.2 Actor 的更新

Actor 用 advantage 更新策略：

\[ \theta \leftarrow \theta + \alpha_\theta \nabla_\theta \log \pi_\theta(A_t\mid S_t) \hat{A}_t \]

9.3 Critic 的更新

Critic 用 TD target 学习价值函数：

\[ y_t=R_{t+1}+\gamma V_\phi(S_{t+1}) \]

价值函数预测是：

\[ V_\phi(S_t) \]

所以 Critic 的 loss 是：

\[ L_V(\phi) = \frac{1}{2} \left( V_\phi(S_t)-y_t \right)^2 \]

即：

\[ L_V(\phi) = \frac{1}{2} \left( V_\phi(S_t) - \left[ R_{t+1}+\gamma V_\phi(S_{t+1}) \right] \right)^2 \]
一句话理解： Actor 负责“怎么做”，Critic 负责“评价做得好不好”。

10. TD Error 与 Advantage 的关系

TD error 定义为：

\[ \delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t) \]

它其实可以看作一步 advantage 的估计。

10.1 从 Advantage 定义出发

\[ A^\pi(s,a)=Q^\pi(s,a)-V^\pi(s) \]

而 \(Q^\pi(s,a)\) 可以写成：

\[ Q^\pi(S_t,A_t) = \mathbb{E} [ R_{t+1}+\gamma V^\pi(S_{t+1}) \mid S_t,A_t ] \]

所以：

\[ A^\pi(S_t,A_t) = \mathbb{E} [ R_{t+1}+\gamma V^\pi(S_{t+1})-V^\pi(S_t) \mid S_t,A_t ] \]

括号里的样本量就是：

\[ \delta_t=R_{t+1}+\gamma V(S_{t+1})-V(S_t) \]
结论： TD error \(\delta_t\) 是 advantage 的一个单步样本估计。

11. GAE 逐步推导

GAE 全称是 Generalized Advantage Estimation，中文可以叫广义优势估计。

它想解决的问题是：

单步 TD 方差小但偏差大；完整 Monte Carlo 回报偏差小但方差大。GAE 用一个参数 \(\lambda\) 在二者之间折中。

11.1 单步 TD Advantage

\[ \hat{A}_t^{(1)} = \delta_t = R_{t+1}+\gamma V(S_{t+1})-V(S_t) \]

11.2 两步 Advantage

两步回报目标是：

\[ R_{t+1} + \gamma R_{t+2} + \gamma^2 V(S_{t+2}) \]

减去当前价值：

\[ \hat{A}_t^{(2)} = R_{t+1} + \gamma R_{t+2} + \gamma^2 V(S_{t+2}) - V(S_t) \]

它也可以写成 TD error 的形式：

\[ \hat{A}_t^{(2)} = \delta_t+\gamma\delta_{t+1} \]

11.3 三步 Advantage

\[ \hat{A}_t^{(3)} = \delta_t+\gamma\delta_{t+1}+\gamma^2\delta_{t+2} \]

11.4 一般 \(n\) 步 Advantage

\[ \hat{A}_t^{(n)} = \sum_{l=0}^{n-1} \gamma^l \delta_{t+l} \]

11.5 GAE 加权平均

GAE 把不同步数的 advantage 加权平均，权重由 \(\lambda\) 控制。最终可以化简为：

\[ \boxed{ \hat{A}_t^{\mathrm{GAE}(\gamma,\lambda)} = \sum_{l=0}^{\infty} (\gamma\lambda)^l \delta_{t+l} } \]

其中：

\[ \delta_t = R_{t+1}+\gamma V(S_{t+1})-V(S_t) \]

11.6 \(\lambda\) 怎么理解？

\(\lambda\) 含义 效果
\(\lambda=0\) 只用单步 TD 方差小，偏差大
\(\lambda=1\) 接近 Monte Carlo 偏差小，方差大
\(0<\lambda<1\) 多步折中 实践中常用
一句话理解 GAE： 它把未来多个 TD error 按 \((\gamma\lambda)^l\) 衰减加起来，得到更稳定的 advantage 估计。

12. Importance Sampling Ratio

PPO 的一个核心问题是：数据是旧策略采样来的，但我们要更新新策略。

假设采样数据来自旧策略：

\[ \pi_{\theta_{\mathrm{old}}}(a\mid s) \]

当前要更新的是新策略：

\[ \pi_\theta(a\mid s) \]

为了衡量新旧策略对同一个动作的概率变化，定义 ratio：

\[ \boxed{ r_t(\theta) = \frac{\pi_\theta(A_t\mid S_t)} {\pi_{\theta_{\mathrm{old}}}(A_t\mid S_t)} } \]

12.1 ratio 的直观含义

情况 含义
\(r_t(\theta)=1\) 新旧策略给这个动作的概率一样。
\(r_t(\theta)>1\) 新策略提高了这个动作的概率。
\(r_t(\theta)<1\) 新策略降低了这个动作的概率。

12.2 普通策略目标

如果直接用 ratio 写策略目标：

\[ L^{\mathrm{PG}}(\theta) = \mathbb{E}_t \left[ r_t(\theta)\hat{A}_t \right] \]

如果 \(\hat{A}_t>0\)，我们希望增大 \(r_t(\theta)\)，也就是提高这个动作概率。

如果 \(\hat{A}_t<0\)，我们希望减小 \(r_t(\theta)\)，也就是降低这个动作概率。

问题： 如果不加限制，策略可能更新过猛，导致新策略和旧策略差得太远，训练崩掉。

13. PPO Clip Objective

PPO 的核心思想是：允许策略变好，但不允许每次更新变化太大。

13.1 Clip ratio

PPO 把 ratio 限制在：

\[ [1-\epsilon,1+\epsilon] \]

例如 \(\epsilon=0.2\)，那么：

\[ r_t(\theta)\in[0.8,1.2] \]

clip 后的 ratio 是：

\[ \mathrm{clip}(r_t(\theta),1-\epsilon,1+\epsilon) \]

13.2 PPO clipped objective

\[ \boxed{ L^{\mathrm{CLIP}}(\theta) = \mathbb{E}_t \left[ \min \left( r_t(\theta)\hat{A}_t, \mathrm{clip}(r_t(\theta),1-\epsilon,1+\epsilon)\hat{A}_t \right) \right] } \]

13.3 为什么有 min？

这个公式初学者很容易卡住。我们分情况理解。

情况一：\(\hat{A}_t>0\)

动作比平均好，应该提高概率。也就是希望：

\[ r_t(\theta)>1 \]

但是不能提高太多。如果 \(r_t(\theta)>1+\epsilon\)，clip 后只算到 \(1+\epsilon\)。

由于 \(\hat{A}_t>0\)，过大的 \(r_t\hat{A}_t\) 会被 min 截断：

\[ \min(r_t\hat{A}_t,(1+\epsilon)\hat{A}_t) = (1+\epsilon)\hat{A}_t \]

意思是：这个好动作可以提高概率，但最多提高到一个范围内。

情况二：\(\hat{A}_t<0\)

动作比平均差，应该降低概率。也就是希望：

\[ r_t(\theta)<1 \]

但是也不能降低太多。如果 \(r_t(\theta)<1-\epsilon\)，clip 后只算到 \(1-\epsilon\)。

注意 \(\hat{A}_t<0\)，乘负数会反向。min 会阻止策略从“过度降低坏动作概率”中获得更多收益。

PPO 的一句话理解： 好动作可以提高概率，但不能一下提高太多；坏动作可以降低概率，但不能一下降低太多。

14. PPO 总损失

实际 PPO 通常不只有 policy loss，还包括 value loss 和 entropy bonus。

14.1 Policy loss

由于深度学习框架通常做最小化，所以最大化 \(L^{\mathrm{CLIP}}\) 等价于最小化：

\[ L_{\mathrm{policy}} = - L^{\mathrm{CLIP}}(\theta) \]

14.2 Value loss

\[ L_{\mathrm{value}} = \frac{1}{2} \left( V_\phi(S_t)-\hat{R}_t \right)^2 \]

其中 \(\hat{R}_t\) 是 value target，例如：

\[ \hat{R}_t=\hat{A}_t+V_\phi(S_t) \]

14.3 Entropy bonus

为了鼓励探索，常加入策略熵：

\[ H(\pi_\theta(\cdot\mid s)) = -\sum_a \pi_\theta(a\mid s) \log \pi_\theta(a\mid s) \]

熵越大，说明策略越分散，探索越多。

14.4 PPO 总目标

常见写法：

\[ L_{\mathrm{total}} = L_{\mathrm{policy}} + c_1L_{\mathrm{value}} - c_2H(\pi_\theta) \]

这里：

项 作用
\(L_{\mathrm{policy}}\) 让策略提高高 advantage 动作的概率。
\(L_{\mathrm{value}}\) 让 Critic 更准确估计价值。
\(-c_2H\) 鼓励策略保持一定探索。

15. 和大模型 RLHF 的对应关系

现在把 Stage 2 的概念映射到大模型后训练。

强化学习概念 大模型中对应什么 解释
策略 \(\pi_\theta(a\mid s)\) 语言模型 给定上下文，输出 token 概率。
状态 \(s\) prompt + 已生成 tokens 模型当前上下文。
动作 \(a\) 下一个 token 或 response 模型生成行为。
轨迹 \(\tau\) 完整生成序列 从 prompt 到 final answer。
Reward 奖励模型分数 / 规则奖励 回答质量、答案正确性、偏好分数。
Critic / Value Model Value Head 估计当前生成状态的未来 reward。
Advantage 回答比 baseline 好多少 PPO 更新的核心权重。
Ratio \(r_t(\theta)\) 新旧模型 token 概率比 控制策略更新幅度。
KL / clip 防止模型偏离 reference model 避免模型为了 reward 训崩。

15.1 LLM PPO 的特殊点

在传统 PPO 中，每一步动作都有一个 reward。但大模型中，很多时候是一整段回答生成完之后才给一个 reward。 

Prompt
  ↓
token1
  ↓
token2
  ↓
...
  ↓
tokenN
  ↓
Reward Model 给整段 response 打分

这会带来 credit assignment 问题：最终 reward 应该如何分配给中间每个 token？

这也是 LLM PPO 难的地方： reward 往往是 sequence-level，但策略更新发生在 token-level。

15.2 为什么需要 KL 约束？

如果只追求 reward，模型可能学会钻奖励模型漏洞，输出奇怪文本。因此 RLHF 中通常加入 KL 惩罚：

\[ \mathrm{KL} \left( \pi_\theta(\cdot\mid s) \| \pi_{\mathrm{ref}}(\cdot\mid s) \right) \]

它的作用是限制当前模型不要偏离 reference model 太远。

一句话： PPO 负责让模型朝高 reward 方向走；KL 负责提醒模型不要走太远。

16. 一页总结

概念 核心公式 你要理解什么
目标函数 \(J(\theta)=\mathbb{E}_{\tau\sim p_\theta(\tau)}[G(\tau)]\) 策略平均能拿多少回报。
log-derivative trick \(\nabla p=p\nabla\log p\) 把轨迹概率求导变成 log 概率求导。
Policy Gradient \(\nabla J=\mathbb{E}[\nabla\log\pi_\theta(A_t\mid S_t)G_t]\) 高回报动作提高概率。
Baseline \(G_t-b(S_t)\) 降低方差，不改变期望梯度。
Advantage \(A^\pi(s,a)=Q^\pi(s,a)-V^\pi(s)\) 动作比平均水平好多少。
Actor-Critic Actor 更新策略，Critic 估计价值 用 value model 辅助策略更新。
GAE \(\hat{A}_t=\sum_l(\gamma\lambda)^l\delta_{t+l}\) 多步 TD error 加权，平衡偏差方差。
PPO Ratio \(r_t=\frac{\pi_\theta(A_t\mid S_t)}{\pi_{\theta_{\mathrm{old}}}(A_t\mid S_t)}\) 衡量新旧策略概率变化。
PPO Clip \(\min(r_t\hat{A}_t,\mathrm{clip}(r_t,1-\epsilon,1+\epsilon)\hat{A}_t)\) 限制策略每次更新幅度。
进入 Stage 3 前的标准： 你应该能解释清楚三件事：
  1. 为什么 \(\nabla_\theta \log \pi_\theta(a\mid s)\) 可以提高某个动作的概率；
  2. 为什么 advantage 比直接用 return 更稳定；
  3. 为什么 PPO 要限制 ratio，不能让策略一次变化太大。
